== NAME

loop - generate numerical or alphabetical sequences for shell loops

== SYNOPSIS

*loop* [-c] _start_ _finish_ [_increment_]

== DESCRIPTION

*loop* generates a sequence of numbers or characters between _start_ and _finish_ (inclusive), outputting one value per line to standard output. This utility is primarily designed for driving loops in shell scripts.

The default _increment_ is `1` (or `-1` if _start_ is greater than _finish_).

=== Preserving Padding

For integer sequences, if _start_ or _finish_ contains leading zeros, *loop* preserves the padding in the output. It uses the maximum field width from the arguments to pad the output values with leading zeros (e.g., `008` to `010`).

== OPTIONS

*-c*::
	Enable character (char) looping. When this option is specified, _start_ and _finish_ must be single characters, and the sequence will progress alphabetically.

== EXAMPLES

.Generate numbers from 10 to 30 in steps of 5
[source,bash]
----
loop 10 30 5
----
Output:
....
10
15
20
25
30
....

.Generate a descending sequence of zero-padded numbers
[source,bash]
----
loop 008 005
----
Output:
....
008
007
006
005
....

.Generate floating-point numbers
[source,bash]
----
loop 0.1 1.6 0.5
----
Output:
....
0.1
0.6
1.1
1.6
....

.Generate an alphabetical sequence of characters from 'a' to 'e'
[source,bash]
----
loop -c a e
----
Output:
....
a
b
c
d
e
....

== BUGS

Character looping only supports single characters, and increments must be integers.
