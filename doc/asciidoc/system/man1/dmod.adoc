== NAME
dmod - apply arithmetic operations to a stream of doubles

== SYNOPSIS

[source]
----
dmod [-a add] [-s sub] [-m mult] [-d div] [-A] [-e exp] [-r root] [doubles]
----


[[description]]
== DESCRIPTION

*dmod* reads a stream of 64-bit double-precision floating-point numbers,
applies one or more arithmetic operations to every value, and writes the
modified doubles to standard output.

Any number of add, subtract, multiply, divide, absolute value, or
exponentiation operations may be given on the command line. They are
applied to each value in the order they appear, so the operations form a
left-to-right pipeline.

If a file argument is supplied, *dmod* reads from it; otherwise it reads
from standard input. Both the input and output are raw binary doubles,
so the standard input and output must not be a terminal. See *dconv*(1)
and *d-i*(1) for converting between doubles and other numeric formats.

[[options]]
== OPTIONS

*-a* _val_::
  Add _val_ to each number.

*-s* _val_::
  Subtract _val_ from each number.

*-m* _val_::
  Multiply each number by _val_.

*-d* _val_::
  Divide each number by _val_. A _val_ of zero is rejected.

*-A*::
  Replace each number with its absolute value.

*-e* _val_::
  Raise each number to the power _val_.

*-r* _val_::
  Take the _val_-th root of each number (raise to the power 1/_val_). A
  _val_ of zero is rejected.

*-h*, *-?*::
  Print a usage message and exit.

[[examples]]
== EXAMPLES

.Rescale a height field to inches and clamp negatives
====
[source,bash]
----
dmod -m 0.0393701 -A terrain_mm.d > terrain_in_pos.d
----
Reads doubles from _terrain_mm.d_ (values in millimeters), multiplies each
by 0.0393701 to convert to inches, then takes the absolute value to remove any
negative elevations, writing the result to _terrain_in_pos.d_. Operations
are applied left to right, so the scaling happens before the absolute value.
====
