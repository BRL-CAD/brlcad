== NAME
ply-g - convert a Stanford PLY file to a BRL-CAD database

== SYNOPSIS

[source]
----
ply-g [-s scale_factor] [-d] [-v] input_file.ply output_file.g
----


[[description]]
== DESCRIPTION

*ply-g* converts the geometry in a Stanford PLY file (ASCII or
binary) into a BRL-CAD _.g_ database via the GCV conversion library. The
resulting BRL-CAD model contains the triangulated mesh described by
_input_file.ply_.

PLY data is conventionally expressed in meters, so by default the input
coordinates are scaled by 1000.0 to produce a model in BRL-CAD's base unit of
millimeters. Use *-s* to override this factor.

[[options]]
== OPTIONS

*-s* _scale_factor_::
  Multiply all input coordinates by _scale_factor_ instead of the default of
  1000.0. For example, a value of 1.0 keeps the data in its original units and
  a value of 500 produces a model half the default size.

*-d*::
  Enable the GCV debug flag, producing additional low-level diagnostic output
  during conversion.

*-v*::
  Run verbosely, reporting progress and details about the conversion.

[[examples]]
== EXAMPLES

.Convert a binary PLY scan and inspect it
====
[source,bash]
----
ply-g bunny.ply bunny.g
mged bunny.g 'tops'
----
Convert the PLY mesh _bunny.ply_ into the database _bunny.g_, then list the
top-level objects with *mged*(1) to verify the imported geometry.
====

.Preserve native units
====
[source,bash]
----
ply-g -s 1.0 model.ply model.g
----
Import _model.ply_ without the default meters-to-millimeters scaling, leaving
the coordinates exactly as stored in the file.
====
