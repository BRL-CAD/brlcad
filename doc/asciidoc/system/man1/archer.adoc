== NAME
archer - graphical user interface for BRL-CAD solid modeling

== SYNOPSIS

[source]
----
archer [_geometry_]
----


[[description]]
== DESCRIPTION

*archer* is a graphical user interface for BRL-CAD, providing an
integrated environment for solid modeling, geometry editing, and ray
tracing. It serves as a modern alternative to the *mged*(1) command-line
modeler and requires Tk to run.

If a geometry argument is given, *archer* opens it on startup. The
argument is normally a BRL-CAD geometry database (a _.g_ file), but it
may also be a file in an importable format such as STL, STEP, or
Wavefront OBJ, in which case the data is converted and loaded into a new
database. When invoked with no argument, *archer* changes to the user's
home directory and starts with an empty workspace; a database can then be
created or opened from the File menu.

[[options]]
== OPTIONS

_geometry_::
  Optional geometry file to open on startup. A BRL-CAD _.g_ database is
  opened directly; a supported foreign format (e.g. _.stl_, _.step_,
  _.obj_) is imported into a new database.

[[examples]]
== EXAMPLES

.Open an existing database
====
[source,bash]
----
archer havoc.g
----
Launches the *archer* interface and opens the existing _havoc.g_
geometry database for editing and visualization.
====

.Import foreign geometry for editing
====
[source,bash]
----
archer part.stl
----
Starts *archer*, converts the _part.stl_ triangle mesh into a BRL-CAD
database, and loads it so the imported geometry can be inspected,
edited, and saved as a native _.g_ file. Compare with the batch
converter *stl-g*(1).
====
