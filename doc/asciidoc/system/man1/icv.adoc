== NAME

icv - convert, compare, and report information for image formats

== SYNOPSIS

[source]
----
icv [-w width] [-n height] [--input-format fmt] [--output-format fmt] [-i input] [-o output] [input] [output]
icv info file [file ...]
icv diff [options] img1 img2
----

[[description]]
== DESCRIPTION

Without a subcommand, *icv* acts as an image format converter, reading the input image and writing it to the output file. Image formats are auto-detected from file extensions unless overridden with *--input-format* / *--output-format*. To stream through standard input or output (a path of which carries no extension) the corresponding format must be given explicitly. This tool is intended to eventually replace the family of single-purpose *pix-png*(1), *png-pix*(1), *bw-pix*(1) and similar converters.

=== Supported Formats

[cols="2*"]
|===
|Format |Extension

|Black and White (Grayscale)
|bw

|Double Precision Pixel (R/G/B/A double)
|dpix

|Pixel Sequence (R/G/B unsigned char)
|pix

|Portable Network Graphics
|png

|Portable Pixmap
|ppm
|===

=== Subcommands

*info*::
	Report the width, height, format, number of channels, color space, and any embedded render information (like the database and objects used to render it) for the specified image file(s).

*diff*::
	Compare two images and report per-channel match statistics on standard error. When the images differ and no output file is given, a raw PIX difference image is written to standard output (only if stdout is not a terminal), matching the behaviour of *pixdiff*(1). Use *-o* to write the diff image to a named file instead.

[[options]]
== OPTIONS

=== Conversion Options

*-i* _file_, *--input* _file_::
	Specify the input file name. If not specified, the first positional argument is used.

*-o* _file_, *--output* _file_::
	Specify the output file name. If not specified, the second positional argument is used.

*-w* _width_, *--width* _width_::
	Specify the width of the input image. Required for raw formats (like PIX and BW) that do not encode dimensional information, unless the size can be automatically deduced from the file size.

*-n* _height_, *--height* _height_::
	Specify the height of the input image. Required for raw formats (like PIX and BW) that do not encode dimensional information.

*--input-format* _fmt_::
	Explicitly set the input image format (e.g., `png`, `pix`, `bw`).

*--output-format* _fmt_::
	Explicitly set the output image format.

*-h*, *--help*::
*-?*::
	Print conversion help and exit.

=== Diff Subcommand Options

*-o* _file_, *--output* _file_::
	Specify a file path to save the diff image output.

*--output-format* _format_::
	Explicitly set the format of the output diff image.

*--format-img1* _format_::
	Explicitly set the format of the first input image.

*--format-img2* _format_::
	Explicitly set the format of the second input image.

*--width-img1* _#_::
	Specify the width of the first input image (raw formats only).

*--height-img1* _#_::
	Specify the height of the first input image (raw formats only).

*--width-img2* _#_::
	Specify the width of the second input image (raw formats only).

*--height-img2* _#_::
	Specify the height of the second input image (raw formats only).

*-h*, *--help*::
*-?*::
	Print diff subcommand help and exit.

[[examples]]
== EXAMPLES

.Convert a raw PIX render to PNG
====
[source,bash]
----
icv -w 1024 -n 512 render.pix render.png
----
Reads a headerless 1024x512 PIX file and writes a PNG. The dimensions are required because PIX (and BW) carry no size information in the file; when omitted, *icv* attempts to deduce them from the file size.
====

.Inspect images before further processing
====
[source,bash]
----
icv info *.png
----
Reports the width, height, format, channel count, color space, and any embedded render information (originating database and objects) for every PNG in the directory, useful for confirming a batch of renders is consistent before compositing.
====

.Regression-test a render against a known-good reference
====
[source,bash]
----
icv diff -o delta.png reference.png current.png
----
Compares two renders, prints per-channel match statistics to standard error, and writes a highlighted difference image to delta.png. Omitting *-o* streams a raw PIX diff to standard output for piping, exactly as *pixdiff*(1) does.
====
