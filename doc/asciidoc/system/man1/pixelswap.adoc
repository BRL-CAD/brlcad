== NAME
pixelswap - interchange two pixel colors in a pix image

== SYNOPSIS

[source]
----
pixelswap [-d _pixeldepth_] _r_ _g_ _b_ _R_ _G_ _B_ < input.pix > output.pix
----


[[description]]
== DESCRIPTION

*pixelswap* reads a stream of RGB *pix*(5) data from standard input and
exchanges two specific colors named on the command line.  Any pixel matching
the first color (_r_ _g_ _b_) is rewritten as the second color (_R_ _G_ _B_),
and any pixel matching the second color is rewritten as the first.  All other
pixels are passed through to standard output unchanged.

Each of the six color components is an integer in the range 0-255.  Both
standard input and standard output must be redirected; *pixelswap* refuses to
run if either is a terminal.

Although a pixel depth may be specified with *-d*, the current implementation
always processes 3-byte RGB pixels and ignores the requested depth.

[[options]]
== OPTIONS

*-d* _pixeldepth_::
  Number of bytes per pixel (must be a positive integer).  Accepted for
  forward compatibility but currently ignored; pixels are always treated as
  3-byte RGB.

*-h*, *-?*::
  Print a usage message and exit.

[[examples]]
== EXAMPLES

.Recolor a rendered frame
====
[source,bash]
----
rt -o frame.pix scene.g region > /dev/null
pixelswap 255 0 0 0 0 255 < frame.pix > swapped.pix
----
Ray traces a model with *rt*(1), then swaps every pure-red pixel for pure-blue
and every pure-blue pixel for pure-red, producing _swapped.pix_.  This is a
quick way to re-key flat region colors without re-rendering the scene.
====
