== NAME
rtweight - calculate mass, volume, and centroid of geometry via ray tracing

== SYNOPSIS

[source]
----
rtweight [options] model.g objects...
----


[[description]]
== DESCRIPTION

*rtweight* fires a grid of rays at the listed _objects_ in the BRL-CAD
database _model.g_ and uses the line-of-sight (LOS) thickness of each
region, combined with per-material density, to compute the volume,
mass, and overall geometric centroid of the geometry.  Boolean
operations are honored, so overlapping or subtracted solids are
accounted for correctly.

Density information is obtained, in order of preference, from a density
file named with *-d*, from a *_DENSITIES* table embedded in _model.g_,
from *material* objects in the database, or finally from a *.density*
file in the current directory.  A density file contains one entry per
line with the integer material id, the density in grams per cubic
centimeter, and the material name, separated by whitespace.  Regions
whose material id has no density entry have their mass reported as
undefined while their volume is still computed.  See *gqa*(1) for more
on defining and managing density information.

By default *rtweight* reports the mass of every region individually
(by region path name and by region id) in addition to the totals; use
*-R* to print only the summary.  Results are written in metric or
English units chosen from the database's editing units.

*rtweight* accepts the same view, grid, and threading options as
*rt*(1); the most useful ones are listed below.  The *-C* (background
color) and *-W* (white background) options of *rt*(1) are disabled
here.

[[options]]
== OPTIONS

*-d* _file_::
  Read density definitions from _file_ instead of from the database.

*-o* _file_::
  Write the report to _file_ rather than to standard output.

*-r*::
  Report the mass of each region verbosely (this is the default).

*-R*::
  Suppress the per-region report and print only the totals.

*-s* _#_::
  Fire a square grid of _#_ by _#_ rays (default 512).  Conflicts with *-g*.

*-g* _#_::
  Set the grid cell width in millimeters.  Conflicts with *-s*.

*-G* _#_::
  Set the grid cell height in millimeters.

*-a* _deg_::
  Set the view azimuth in degrees.  Use with *-e*; conflicts with *-M*.

*-e* _deg_::
  Set the view elevation in degrees.  Use with *-a*; conflicts with *-M*.

*-M*::
  Read the model2view matrix from standard input.  Conflicts with *-a* and *-e*.

*-P* _#_::
  Use at most _#_ processor cores.  The default is all available cores;
  a negative value means use all but that many.

*-U* _#_::
  Set whether air-coded regions are retained (0 = discard, the default; 1 = keep).

*-h*, *-?*::
  Print a help summary and exit.

Most other *rt*(1) options (non-square views, perspective, tolerances,
debugging flags, and so on) are also accepted.

[[examples]]
== EXAMPLES

.Total mass and centroid with a custom density file
====
[source,bash]
----
rtweight -d steel.density -R -s 1024 engine.g block crankshaft > engine.wt
----
Fires a 1024x1024 grid at the _block_ and _crankshaft_ regions of
_engine.g_, using the material densities defined in _steel.density_,
and writes only the volume, mass, and centroid totals to _engine.wt_.
====

.Per-region breakdown using densities stored in the database
====
[source,bash]
----
rtweight -a 35 -e 25 model.g hull
----
Ray traces the _hull_ assembly from azimuth 35, elevation 25 degrees,
pulling density values from the *_DENSITIES* table (or *material*
objects) inside _model.g_, and prints the mass of each contained region
followed by the overall totals.
====
