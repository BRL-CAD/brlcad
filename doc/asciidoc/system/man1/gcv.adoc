== NAME
gcv - geometry format converter

== SYNOPSIS

[source]
----
gcv [options] [--input-format _fmt_] [--output-format _fmt_] _input_file_ _output_file_
----

[source]
----
gcv [options] -i _input_file_ -o _output_file_
----


[[description]]
== DESCRIPTION

*gcv* is the unified BRL-CAD geometry converter. It reads a model in one
file format and writes it out in another, selecting the appropriate read
and write filter plugins from *libgcv* based on the file format. Supported
formats include the BRL-CAD database (*.g*), STEP, IGES, STL, and Wavefront
OBJ, among others.

The input and output files may be given as the last two positional
arguments, or named explicitly with *-i* and *-o*. When only one of *-i* or
*-o* is used, the remaining positional argument supplies the other file.

The file format is normally inferred from the file extension. It may instead
be stated with *--input-format* and *--output-format*, or with a multi-letter
format prefix on the path itself in the form _fmt_:_path_ (e.g.
*step:model.stp*). A single-letter prefix is not recognized as a format, so
that Windows drive letters are not misinterpreted.

Per-filter options may be passed through to the read filter, the write
filter, or both, allowing format-specific behavior to be tuned without
*gcv* needing to know about each option.

[[options]]
== OPTIONS

*-h*, *--help* [_format_]::
  Print help and exit. If a _format_ is named, format-specific help is
  intended to be printed for it.

*-?* [_format_]::
  Same as *-h*.

*-i* _file_, *--input* _file_::
  Read geometry from _file_. The file must exist.

*-o* _file_, *--output* _file_::
  Write geometry to _file_. If _file_ already exists, conversion output is
  appended where the write filter supports it.

*--input-format* _format_::
  Set the input file format explicitly (e.g. *step*, *iges*, *stl*, *obj*),
  overriding extension- and prefix-based detection.

*--output-format* _format_::
  Set the output file format explicitly.

*-I* _opts_, *--input-only-opts* _opts_::
  Pass _opts_ only to the input (read) filter. Consumes arguments until the
  next top-level *gcv* option is encountered.

*-O* _opts_, *--output-only-opts* _opts_::
  Pass _opts_ only to the output (write) filter. Consumes arguments until the
  next top-level *gcv* option is encountered.

*-B* _opts_, *--input-and-output-opts* _opts_::
  Pass _opts_ to both the input and output filters. Consumes arguments until
  the next top-level *gcv* option is encountered.

[[examples]]
== EXAMPLES

.Convert a STEP file into a BRL-CAD database
====
[source,bash]
----
gcv assembly.stp assembly.g
----
Reads the STEP file _assembly.stp_ and writes a BRL-CAD database
_assembly.g_. Both formats are detected from the file extensions, after
which the result can be inspected with *mged*(1) or rendered with *rt*(1).
====

.Tune a filter while converting OBJ to a .g database
====
[source,bash]
----
gcv -I --units mm model.obj model.g
----
Converts a Wavefront OBJ mesh to a BRL-CAD database, passing the
*--units mm* option only to the OBJ read filter so that the source
coordinates are interpreted as millimeters.
====

.Override detection with explicit formats
====
[source,bash]
----
gcv --input-format step --output-format stl in.dat out.dat
----
Forces the input to be read as STEP and the output written as STL even
though the *.dat* extensions give no hint of the formats involved.
====
