== NAME
showshot - generate MGED commands representing a ray through some geometry

== SYNOPSIS

[source]
----
showshot [-n name] [-r radius] [-?] < input
----


[[description]]
== DESCRIPTION

*showshot* reads the description of a ray through some geometry on standard
input and writes *mged*(1) commands to standard output that build a database
object representing that ray. Each line of input must have the form

....
object  x_in  y_in  z_in  x_out  y_out  z_out
....

as produced by *nirt*(1), where _object_ is the name of a database object along
the ray and (__x_in__, __y_in__, __z_in__) and (__x_out__, __y_out__,
__z_out__) are the ray's entry and exit points for that object. The first and
last lines of input must correspond to the first and last objects encountered
along the ray; the intervening lines may appear in any order.

*showshot* emits commands to create an RCC whose vertex is the entry point of
the first object and which extends to the exit point of the last object (all
other coordinates are ignored). It then makes the RCC a member of a region
colored yellow (255 255 0) and creates a group containing the ray region along
with every object named in the input.

[[options]]
== OPTIONS

*-n* _name_::
  Use _name_ as the root for the generated object names: the ray solid is
  _name_.s, the ray region is _name_.r, and the group is _name_. The default is
  ray._PID_, where _PID_ is the process ID of *showshot*.

*-r* _radius_::
  Set the radius of the RCC representing the ray, in database units. The
  default is 1.

*-?*::
  Print a usage message and exit.

[[examples]]
== EXAMPLES

.Visualize a NIRT shotline in MGED
====
[source,bash]
----
nirt -b -f entexit.nrt model.g component > shot.txt
showshot -n shotline -r 0.5 < shot.txt > shot.mged
mged model.g < shot.mged
----
Fire a ray through _component_ with *nirt*(1) using a format that prints the
entry and exit points of each partition, convert that report into a thin
(radius 0.5) yellow ray named _shotline_ with *showshot*, then source the
generated commands back into *mged*(1) to draw the ray alongside the model.
====
