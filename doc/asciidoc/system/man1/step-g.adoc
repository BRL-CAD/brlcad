== NAME
step-g - convert a STEP (ISO 10303 AP203) file to a BRL-CAD geometry database

== SYNOPSIS

[source]
----
step-g [-v] [-D] [-S _summary_file_] [-e _ids_]
       (-o _outfile.g_ | -O _outfile.g_) _infile.stp_
----


[[description]]
== DESCRIPTION

*step-g* reads an ISO 10303 STEP (Standard for the Exchange of Product
model data) file, typically an AP203 _.stp_ or _.step_ file, and translates
its boundary representation (B-rep) geometry and assembly hierarchy into a
BRL-CAD _.g_ database.

The translation maps STEP manifold solids and bounded surfaces into BRL-CAD
*brep* objects and preserves the structure of assemblies and parts using
BRL-CAD combinations. As input, _infile.stp_ may be a filename or *-* to read
the STEP data from standard input.

An output file must be named with either *-o* or *-O*. Load and convert times
along with a brief translation summary are written to standard error.

[[options]]
== OPTIONS

*-o* _outfile.g_::
  Name the BRL-CAD geometry database to create. If the file already exists,
  *step-g* refuses to overwrite it and exits with an error.

*-O* _outfile.g_::
  Name the BRL-CAD geometry database to create, forcibly overwriting the file
  if it already exists.

*-S* _summary_file_::
  Write a detailed translation summary log to the named file.

*-v*::
  Enable verbose diagnostic output to standard error during parsing and
  conversion.

*-D*::
  Perform a dry run: load and parse the STEP file and exercise the conversion
  steps, but do not write the output _.g_ database. Useful for validating a
  STEP file or timing the parsing phase.

*-e*, *--entity* _IDS_::
  Convert only the listed STEP representation-item roots while retaining their
  product and assembly context. _IDS_ is a comma-separated list of positive
  Part 21 instance identifiers, each optionally prefixed by *#*. The option
  may be repeated. Dependencies required by a selected root are still loaded;
  leaf faces, edges, and points are not independently convertible roots.

[[examples]]
== EXAMPLES

.Convert a STEP file and inspect the result
====
[source,bash]
----
step-g -o widget.g widget.stp
mged widget.g 'tops'
----
Translates _widget.stp_ into the new database _widget.g_ (refusing to clobber
an existing _widget.g_), then lists the top-level objects of the resulting
database with *mged*(1) so the imported assembly tree can be examined.
====

.Validate a STEP file without writing output
====
[source,bash]
----
step-g -v -D -o /dev/null suspect.step
----
Loads and parses _suspect.step_ with verbose diagnostics enabled to check the
file's integrity and time the parse, while the *-D* dry run suppresses writing
any geometry.
====
