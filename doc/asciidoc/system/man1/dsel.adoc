== NAME
dsel - selectively keep or skip doubles from a data stream

== SYNOPSIS

[source]
----
dsel _keep_ < input.doubles > output.doubles
dsel _skip_ _keep_ [_skip_ _keep_ ...] < input.doubles > output.doubles
----


[[description]]
== DESCRIPTION

*dsel* reads a stream of double-precision floating-point numbers from
standard input, selectively copies or discards runs of values according to
its positional arguments, and writes the kept values to standard output.
Both input and output must be redirected; *dsel* refuses to run if either
standard input or standard output is a terminal.

If a single argument is given, *dsel* copies exactly that many _keep_
values to the output and exits.

With two or more arguments, they are interpreted as alternating _skip_ and
_keep_ counts: *dsel* reads and discards _skip_ values, then reads and
copies _keep_ values, repeating for each pair. If the argument list ends
with an unpaired _skip_ (an odd argument count greater than one), the final
implied _keep_ is treated as infinite, so all remaining values are copied.
A _skip_ count of zero or less is ignored, while a _keep_ count of zero or
less causes *dsel* to stop. If the stream ends before a requested count is
satisfied, *dsel* stops at end-of-input.

This is the double-precision counterpart of the byte- and short-oriented
stream selectors used throughout the BRL-CAD signal-processing tools, and
pairs well with *dconv*(1), *dmod*(1), and *dstats*(1).

[[options]]
== OPTIONS

*-h*, *-?*::
  Print a usage message and exit.

*skip*::
  Number of double-precision values to read and discard.

*keep*::
  Number of double-precision values to read and copy to the output.

[[examples]]
== EXAMPLES

.Extract a window of samples
====
[source,bash]
----
dsel 512 1024 < data.d > window.d
----
Skips the first 512 doubles of _data.d_ (for example, a header or settling
region) and copies the next 1024 doubles to _window.d_, ignoring the rest.
====

.Take a leading block and scale it inline
====
[source,bash]
----
dsel 1024 < signal.d | dmod -m 2.0 > head_scaled.d
----
Copies only the first 1024 doubles of _signal.d_ (a single argument is always
a _keep_ count), then pipes them through *dmod*(1) to scale each value by 2.0
before writing _head_scaled.d_.
====
