== NAME

imgdims - guess the width and height of an image file

== SYNOPSIS

*imgdims* [-n | -s] [-# _bytes_per_pixel_] _file_name_

*imgdims* [-# _bytes_per_pixel_] _num_bytes_

== DESCRIPTION

*imgdims* guesses the dimensions (width and height in pixels) of an image. The image size may be specified either implicitly by providing a file name, or explicitly by specifying the total number of bytes. It uses the autosizing capabilities of `libbn` to make the guess.

== OPTIONS

*-n*::
	Extract the dimensions directly from the file name if it contains standard BRL-CAD resolution indicators (e.g. `1024x768` or `512`). If this fails, the command will exit with an error.

*-s*::
	Ignore any dimensions encoded in the file name and use the file size in bytes to determine the pixel count and guess the width and height.

*-#* _bytes_per_pixel_::
	Specify the size of each pixel in bytes. Common values are `1` (for black and white images) and `3` (for RGB images, which is the default). If this option is omitted, *imgdims* attempts to determine the pixel size based on the file extension (e.g. `.bw` defaults to 1, `.pix` defaults to 3). If no recognized extension is found, it defaults to 3.

*-h*::
*-?*::
	Print usage information and exit.

== EXAMPLES

.Guess the dimensions of a file named 'image.pix' from its file size
[source,bash]
----
imgdims image.pix
----

.Extract dimensions from a filename containing resolution indicators
[source,bash]
----
imgdims -n image_1024x768.bw
----

.Guess the dimensions of a black-and-white image given its file size in bytes
[source,bash]
----
imgdims -# 1 262144
----
