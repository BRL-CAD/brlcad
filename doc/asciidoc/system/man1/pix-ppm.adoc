== NAME

pix-ppm - convert a BRL-CAD PIX or BW image to a PPM/PGM image

== SYNOPSIS

[source]
----
pix-ppm [-a] [-# _bytes_per_pixel_] [-s _squaresize_] [-w _file_width_] [-n _file_height_] [-o _file.ppm_] [_file.pix_] [> _file.ppm_]
----


[[description]]
== DESCRIPTION

*pix-ppm* converts a *pix*(5) (or *bw*(5)) format image into a Netpbm
PPM/PGM format image. Three-byte-per-pixel RGB input is written as a
binary PPM (P6); single-byte-per-pixel input is written as a binary PGM
(P2). Input is read from the named _file.pix_ or, if no file is given,
from standard input; output is written to standard output by default.

PIX and BW files carry no embedded dimensions, so the size of the input
must be supplied (default 512x512) via *-s*, *-w*/*-n*, or determined
automatically with *-a*.


[[options]]
== OPTIONS

*-a*::
  Autosize the input file, inferring the image width and height from the
  file length. Only effective when reading from a named file rather than
  standard input.

*-#* _bytes_per_pixel_::
  Set the number of bytes per pixel in the input. Must be 3 (RGB, the
  default) or 1 (grayscale).

*-s* _squaresize_::
  Set both the input width and height to _squaresize_ pixels.

*-w* _file_width_::
  Set the input width to _file_width_ pixels.

*-n* _file_height_::
  Set the input height to _file_height_ scanlines.

*-o* _file.ppm_::
  Write the image to the named output file. If standard output is also
  redirected or piped, *pix-ppm* writes the same image to both
  destinations. *pix-ppm* refuses to write binary image data to a
  terminal.


[[examples]]
== EXAMPLES

.Convert an autosized PIX rendering to PPM
====
[source,bash]
----
pix-ppm -a render.pix -o render.ppm
----
Reads _render.pix_, infers its dimensions from the file size, and writes
a binary PPM file _render.ppm_ suitable for viewing or importing into
common image tools.
====

.Render and convert in a single pipeline
====
[source,bash]
----
rt -s 1024 -o /dev/stdout model.g component | pix-ppm -s 1024 > component.ppm
----
Pipes a 1024x1024 *rt*(1) raytrace directly into *pix-ppm*, producing a
PPM image without writing an intermediate PIX file to disk.
====
