== NAME
imod - modify a stream of short integers with arithmetic operations

== SYNOPSIS

[source]
----
imod [-a add] [-s sub] [-m mult] [-d div] [-A] [-e exp] [-r root] [file.s] > output.s
----


[[description]]
== DESCRIPTION

*imod* reads a stream of 16-bit signed short integers and applies a
sequence of arithmetic operations to each value, writing the modified
short integers to standard output. If the optional _file.s_ argument is
given, *imod* reads from that file; otherwise it reads from standard
input.

Any number of operations may be given on the command line, and they are
applied to each value in the order they appear. For efficiency the
operations are precomputed once into a 65536-entry lookup table that maps
every possible input short to its result. Because the output is
constrained to the signed 16-bit range (-32768 to 32767), results outside
that range are clipped to the nearest limit and a count of high and low
clipping events is printed to standard error when the program finishes.

*imod* refuses to run if either standard input or standard output is a
terminal. It is the short-integer counterpart of *bwmod*(1) and *dmod*(1),
operating on the same kind of data used by tools such as *dsp*(5).

[[options]]
== OPTIONS

*-a* _val_::
  Add _val_ to each value.

*-s* _val_::
  Subtract _val_ from each value.

*-m* _val_::
  Multiply each value by _val_.

*-d* _val_::
  Divide each value by _val_. A _val_ of zero aborts the program.

*-A*::
  Replace each value with its absolute value.

*-e* _val_::
  Raise each value to the power _val_ (exponentiation).

*-r* _val_::
  Take the _val_-th root of each value (equivalent to raising to the
  power 1/_val_). A _val_ of zero aborts the program.

*-h*, *-?*::
  Print a usage message and exit.

[[examples]]
== EXAMPLES

.Scale and offset a short-integer data stream
====
[source,bash]
----
imod -m 2.0 -a 100 < terrain.s > brighter.s
----
Reads short integers from _terrain.s_, multiplies each by 2.0 and then
adds 100, writing the result to _brighter.s_. The operations are applied
in command-line order, so each value becomes 2*value + 100; any results
that exceed the 16-bit range are clipped and reported on standard error.
====

.Rectify and attenuate a signal before further processing
====
[source,bash]
----
imod -A -d 4 < signal.s | dpeak > peaks.s
----
Takes the absolute value of every sample in _signal.s_, divides each by
4 to bring the magnitudes down, and pipes the rectified stream directly
into *dpeak*(1) for peak detection.
====
