== NAME
mged - multi-display interactive combinatorial solid geometry editor

== SYNOPSIS

[source]
----
mged [options] [database.g] [mged_command]
----


[[description]]
== DESCRIPTION

_MGED_, a.k.a. the "Multi-Device Geometry Editor", is the central interaction environment for the BRL-CAD package. It offers both command line and menu driven interactions, as well as wireframe displays and integrated raytracing capabilities. It is the primary tool for BRL-CAD geometry creation and interactive manipulation.

_MGED_'s history traces back to the earlier "GED" (a.k.a. "Geometry Editor"), and much of GED's functionality is preserved in _MGED_. A full description of MGED's abilities is beyond the scope of this document - the reader is referred to Volumes II and III of the BRL-CAD Tutorial Series, available at http://brlcad.org, for a more comprehensive introduction. This document will deal with the various options available when it comes to starting MGED.

The primary mode for modern _MGED_ usage is the Tcl/Tk based environment, which offers menu support and a number of Tcl based tools. It is also possible to run more minimalist environments of terminal + framebuffer and a terminal without any display window. _MGED_ also allows users to execute single commands from the command line without starting any environment, allowing for activities such as scripting. Some examples of such usage are present in the Examples section of this document.

Technically, _MGED_ can be opened without a database file being supplied, but until one _is_ supplied it will not be terribly useful. This can be done either by supplying a file name on the command line or by opening a database from within _MGED_ using the File->Open... menu or the *opendb* command.

Options that affect per-session appearance (variables, grid, colour) are applied _after_ any `.mgedrc` file is processed, so command-line flags always take precedence over saved settings.


[[options]]
== OPTIONS

=== Display and Mode

*-a* _type_, *--attach* _type_::
Specifies a display manager to automatically attach to when starting _MGED_.
+
Without specifying this option, _MGED_ will prompt which display manager to attach during startup: `attach (nu|X|ogl)[nu]?`
+
The list of available display managers varies from platform to platform and across releases. Commonly available options include the _nu_ (NULL) display manager (starts without any graphical display), the _X_ display manager (X11 wireframe display), and the _ogl_ display manager (OpenGL wireframe display).

*-d* _display_, *--display* _display_::
Specifies which X server to connect to on platforms that have a running X11 server available. This option is an alternative to setting the `DISPLAY` environment variable. You can specify a local or remote display in X11 HOST:PORT format (e.g., `-d :0` or `-d host:0`).

*--dm-type* _type_::
Sets the display manager type to use (e.g., `ogl`, `X`, `swrast`). This corresponds to the `mged_default(dm_type)` Tcl variable used during GUI initialisation.

*--geom* _WxH+X+Y_::
Sets the command window geometry (position and size) in standard X geometry format (e.g., `--geom 800x600+0+0`).

*--ggeom* _WxH+X+Y_::
Sets the graphics/geometry window geometry in standard X geometry format.

=== Run Mode

*-b*, *--background*::
Runs _MGED_ as a background process, freeing the terminal from which it was started.

*-c*, *--classic*::
Instructs _MGED_ to start using classic mode instead of the default Tcl/Tk-based GUI. The *-c* option is often used when specifying an *mged_command* to indicate that _MGED_ is being run in a command-mode without a default GUI.

*-C*, *--gui*::
Instructs _MGED_ to start in GUI (non-classic) mode. This is the default on platforms that support graphics.

*-p*, *--pipe*::
Enables pipe mode. In this mode _MGED_ emits a `MGED_CMD_DONE <status>` sentinel to stdout after every command completes on stdin, making it suitable for use as a subprocess driven by an automated tool.

*-r*, *--read-only*::
Opens the `database.g` file in read-only mode.

=== rc File Control

*--rcfile* _file_::
Loads _file_ instead of the default `.mgedrc` during startup. This overrides the `MGED_RCFILE` environment variable.

*--no-rc*::
Skips loading any `.mgedrc` file entirely, providing a clean-slate session unaffected by personal settings. Useful for automated testing and scripted invocations.

=== Variable Override Escape Hatches

*--set* _VAR=VALUE_::
Sets an MGED variable by name after `.mgedrc` is processed (e.g., `--set use_air=1`). This provides access to all 31 `mged_vparse[]` variables not exposed as individual long options. May be repeated.

*--rset* _ARGS_::
Passes arguments directly to the MGED `rset` command after `.mgedrc` is processed (e.g., `--rset "ax draw 1"`). Provides access to axes, full colour scheme, rubber-band, and any other `rset` resource. May be repeated.

=== Wireframe Appearance

*--linewidth* _#_::
Sets the wireframe line width in pixels (must be >= 1).

*--linestyle* _s|d_::
Sets the line style: `s` for solid, `d` for dashed.

*--faceplate*, *--no-faceplate*::
Shows or hides the faceplate overlay.

*--dlist*, *--no-dlist*::
Enables or disables display lists for wireframe rendering.

=== View and Interaction

*--perspective* _#_::
Sets the perspective angle in degrees. Use `-1` to disable perspective (orthographic view).

*--eye-sep-dist* _#_::
Sets the stereo eye separation distance in millimeters. Use `0` for monoscopic (non-stereo) rendering.

*--coords* _m|v_::
Sets the constraint coordinate system: `m` for model space, `v` for view space.

*--rotate-about* _m|v|e_::
Sets the center of rotation: `m` for model origin, `v` for view center, `e` for eye point.

*--transform* _v|a|e_::
Sets the mouse transform mode: `v` for view, `a` for adc (angle/distance cursor), `e` for edit.

*--mouse-behavior* _v|a|e_::
Sets the mouse behavior mode.

*--perspective-mode* _0|1_::
Enables (`1`) or disables (`0`) perspective mode.

*--sliders* _0|1_::
Shows (`1`) or hides (`0`) the slider bars.

*--context* _0|1_::
Enables (`1`) or disables (`0`) context-sensitive editing mode.

=== Air and Geometry Settings

*--use-air*, *--no-use-air*::
Enables or disables the use_air flag (controls whether air regions are included in ray-trace intersection lists).

*--nmg-eu-dist* _#_::
Sets the NMG edge-use distance tolerance.

=== Framebuffer

*--fb*, *--no-fb* (via --set)::
Use `--set fb=1` or `--set fb=0` to enable or disable framebuffer overlay.

*--fb-overlay* _0|1|2_::
Sets the framebuffer overlay mode: `0` (framebuffer under geometry), `1` (interleaved), `2` (framebuffer over geometry).

*--port* _#_::
Sets the port number for the framebuffer server (0-65535).

=== View Predictor

*--predictor*, *--no-predictor*::
Enables or disables the view predictor (ghost image of predicted view during rotation).

*--predictor-advance* _#_::
Sets the predictor advance time in seconds.

*--predictor-length* _#_::
Sets the predictor trail length in seconds.

=== Grid

*--grid-draw* _0|1_::
Shows (`1`) or hides (`0`) the grid overlay.

*--grid-snap* _0|1_::
Enables (`1`) or disables (`0`) snap-to-grid.

*--grid-rh* _#_::
Sets the horizontal grid resolution (model units).

*--grid-rv* _#_::
Sets the vertical grid resolution (model units).

*--grid-mrh* _#_::
Sets the number of minor grid intervals per major horizontal grid line.

*--grid-mrv* _#_::
Sets the number of minor grid intervals per major vertical grid line.

=== Colour

Colour values may be specified as three space-separated integers 0–255 (`R G B`), as a slash-separated triplet (`R/G/B`), or as an HTML hex colour (`#RRGGBB`).

*--bg* _R G B_::
Sets the background colour.

*--geo-color* _R G B_::
Sets the default geometry wireframe colour.

=== Debugging

*-x* _#_, *--rt-debug* _#_::
Sets the debug level for _librt_ (hexadecimal integer).

*-X* _#_, *--bu-debug* _#_::
Sets the debug level for _libbu_ (hexadecimal integer).

=== Information

*-v*, *--version*::
Displays version information and exits.

*-h*, *-?*, *--help*::
Prints a brief help statement showing available options and exits.


[[rc_file]]
== RC FILE

_MGED_ searches for a startup configuration file (`.mgedrc`) in three locations, in order:

. If the environment variable `MGED_RCFILE` is set, that path is used.
. If the environment variable `HOME` is set, `$HOME/.mgedrc` is used.
. If `.mgedrc` exists in the current working directory, it is used.

The first file found is used and the others are ignored. Use *--rcfile* to specify an explicit path, or *--no-rc* to skip rc file processing entirely.

Command-line options that affect MGED variables (all the long options listed above) are applied _after_ the rc file is processed, so they always take precedence.


[[scripting]]
== SCRIPTING (BATCHING) MGED

Often it is convenient to generate a series of _MGED_ commands outside _MGED_ and then submit the command list to _MGED_, which will react as if each command were being manually entered within it. Simply create a file (say, "cmds.txt") with each line being a desired command, and then have _MGED_ execute the list. At the normal shell prompt, execute


----

$ mged -c tgm.g < cmds.txt
      
----

Note that the procedure described above is very useful in rapid prototyping or automated TGM processing without resorting to low-level library programming in C or C++.

For fully non-interactive scripted use, the *--no-rc* flag avoids loading any personal `.mgedrc` that could affect the session:

----

$ mged --no-rc -c --use-air --linewidth 2 model.g 'some_command arg'

----

[[examples]]
== EXAMPLES

The following are run from the operating system command prompt:

.Running the _MGED_ *l* from the operating system command line.
[example]
====
user:~**mged /usr/brlcad/share/brlcad/7.12.2/db/m35.g l component**


....

Opened in READ ONLY mode
component:  --
   u bed
   u frame
   u cab
   u suspension
   u misc
   u power.train
	
....



====

.Running _MGED_ without a display manager.
[example]
====
user:~**mged -c**


....

BRL-CAD Release 7.13.0  Geometry Editor (MGED)
    Thu, 09 Oct 2008 22:31:41 -0400, Compilation 1
    user@localhost:/usr/brlcad/

attach (nu|X|ogl)[nu]? nu
mged>
	
....



====

.Running _MGED_ and bypassing the attach prompt by specifying the X11 display manager.
[example]
====
user:~**mged -a X -c**


....

BRL-CAD Release 7.15.0  Geometry Editor (MGED)
    Tue, 22 Sep 2009 12:40:01 -0400, Compilation 1
    user@localhost:/usr/brlcad/

ATTACHING X (X Window System (X11))

mged>
	
....



====

.Running _MGED_ with a custom background colour and no personal rc file (scripted use).
[example]
====
user:~**mged --no-rc --bg 0 0 64 --use-air -c model.g**
====

.Setting an MGED variable not exposed as a dedicated option.
[example]
====
user:~**mged --set eye_sep_dist=65 model.g**
====

.Setting a grid resource via the rset escape hatch.
[example]
====
user:~**mged --rset "g snap 1" --rset "g rh 10" model.g**
====
