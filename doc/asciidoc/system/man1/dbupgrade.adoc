== NAME
dbupgrade - upgrade a BRL-CAD mged geometry database to the current version.

== SYNOPSIS

[source]
----
dbupgrade [-f] [-r] [-h] [-?] input.g output.g
----


[[description]]
== DESCRIPTION

*dbupgrade* upgrades an older BRL-CAD geometry database (_input.g_) to the current database version (v5) and writes the result to _output.g_.

The v5 database format is platform-independent, so upgrading eliminates the need to use *g2asc*(1) and *asc2g*(1) to transfer databases between different computer architectures. If an older v4 database was already transferred from a binary-incompatible platform, *dbupgrade* attempts to automatically fix any endianness issues, though this process may be imperfect. For best results, upgrade the database on its original platform before transferring it.

Additionally, *dbupgrade* performs automatic conversions of obsolete geometry types and properties. For example, it converts heightfields (HF) to displacement maps (DSP), polysolids to bag-of-triangles (BOT), and BSPLINEs to boundary representations (BREP). It also updates old material shader properties (e.g., replacing "plastic" with "phong") and migrates material color tables to the `regionid_colortable` global attribute.


[[options]]
== OPTIONS

*-f*::
Force the conversion process. If the input database is already at the current version (v5), the tool normally halts with a warning. Using this option forces the conversion to proceed (e.g., from v5 to v5, or v4 to v4 if reverting).

*-r*::
Revert the database to an older format (currently version 4).

*-h, -?*::
Print a usage message and exit.


[[examples]]
== EXAMPLES

dbupgrade _old_model.g new_model.g_

[quote]
____
Upgrades the older database `old_model.g` to the current format and saves it as `new_model.g`.
____

dbupgrade -f _current_model.g rewritten_model.g_

[quote]
____
Forces the conversion of `current_model.g` even if it is already at the v5 format, which is useful for applying internal geometry updates (like converting polysolids to BOTs).
____

[[diagnostics]]
== DIAGNOSTICS

Numerous error conditions are possible, usually due to errors in the geometry database. Descriptive messages are printed on standard error (file descriptor 2).
