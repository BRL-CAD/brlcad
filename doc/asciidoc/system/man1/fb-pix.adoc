== NAME
fb-pix - save a framebuffer image as a color pix file

== SYNOPSIS

[source]
----
fb-pix [-i] [-c] [-F framebuffer] [-s squaresize] [-w width] [-n height] [-h] [-?] [file.pix]
----

[[description]]
== DESCRIPTION

*fb-pix* reads a region of a framebuffer and saves it as a
*pix*(5) format file. If no output file is given, data is written
to standard output.

By default, the output and framebuffer size are 512x512 pixels,
with a first-quadrant coordinate system (origin at the lower left).

[[options]]
== OPTIONS

*-i*::
Use a fourth-quadrant coordinate system (origin at the upper left),
reading the framebuffer scanlines from top to bottom instead of
bottom to top.

*-c*::
Apply the framebuffer color map to pixel values before writing.

*-F* _framebuffer_::
Specify the framebuffer device. Defaults to the `FB_FILE`
environment variable (see *brlcad*(1)).

*-s* _squaresize_::
Set both width and height to _squaresize_.

*-w* _width_::
Set the output width in pixels.

*-n* _height_::
Set the output height in pixels.

*-h, -?*::
Print a usage message and exit.

[[examples]]
== EXAMPLES

.Capture a rendered image and convert it to PNG
====
[source,bash]
----
$ rt -s 1024 -F /dev/fb havoc.g havoc
$ fb-pix -s 1024 | pix-png -s 1024 > havoc.png
----
Renders a model to the framebuffer with *rt*(1), saves the 1024x1024
framebuffer contents to a *pix*(5) stream on standard output, and pipes
it into *pix-png*(1) to produce a PNG file.
====

.Save the framebuffer color map baked into the pixels
====
[source,bash]
----
$ fb-pix -c -F /dev/ogl scene.pix
----
Saves the image from the `/dev/ogl` framebuffer with the framebuffer
color map applied to each pixel value, so the stored file reflects the
displayed image even if a non-linear color map was loaded.
====
