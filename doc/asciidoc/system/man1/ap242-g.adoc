== NAME
ap242-g - convert a STEP AP242 format file to a BRL-CAD geometry database

== SYNOPSIS

[source]
----
ap242-g [-v] [-D] [-S _summary_file_] (-o _outfile.g_ | -O _outfile.g_) _infile.stp_
----


[[description]]
== DESCRIPTION

*ap242-g* reads an ISO 10303-242 STEP (Standard for the Exchange of Product
model data) AP242 `.stp` or `.step` file and translates its boundary
representation (B-rep) geometry and assembly structure into a BRL-CAD `.g`
database. AP242 (Managed Model Based 3D Engineering) supersedes and merges the
older AP203 and AP214 protocols handled by *step-g*(1).

The translation maps STEP manifolds and bounded surface geometry into BRL-CAD
Boundary Representation (BREP) objects, and preserves the assembly and part
hierarchy using BRL-CAD combinations. Progress, along with elapsed load and
conversion times, is reported to standard error.

The input STEP file may be given as a path, or as *-* to read from standard
input. By default *ap242-g* refuses to overwrite an existing output file.

[[options]]
== OPTIONS

*-o* _outfile.g_::
  Name of the BRL-CAD geometry database to create. If the file already exists,
  *ap242-g* refuses to overwrite it and exits with an error.

*-O* _outfile.g_::
  Name of the BRL-CAD geometry database to create, forcibly overwriting it if it
  already exists.

*-S* _summary_file_::
  Write a detailed summary log of the translation, including parsed entities,
  skipped unsupported types, and overall conversion statistics, to the named file.

*-v*::
  Enable verbose output, printing detailed diagnostic messages to standard error
  during parsing and conversion.

*-D*::
  Perform a dry run. The STEP file is loaded and parsed and the conversion is
  simulated, but no `.g` file is written. Useful for validating a STEP file or
  timing the parse phase.

[[examples]]
== EXAMPLES

.Convert an AP242 part and inspect it
====
[source,bash]
----
ap242-g -o bracket.g bracket.stp
mged bracket.g 'tops'
----
Converts the AP242 file _bracket.stp_ into _bracket.g_ (refusing to clobber an
existing _bracket.g_), then lists the resulting top-level objects with
*mged*(1) so the imported assembly can be examined and raytraced.
====

.Validate a file with a logged dry run
====
[source,bash]
----
ap242-g -v -D -S report.txt -o /dev/null model.step
----
Loads and parses _model.step_ with verbose diagnostics, writes a full
translation summary to _report.txt_, and skips writing geometry. This checks
that an AP242 file converts cleanly before committing to a full run.
====
