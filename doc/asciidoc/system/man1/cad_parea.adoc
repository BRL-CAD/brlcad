== NAME
cad_parea - compute the area of a polygon

== SYNOPSIS

[source]
----
cad_parea [-i input] [-o output]
----


[[description]]
== DESCRIPTION

*cad_parea* reads a series of 2D polygon vertex coordinates (_x_ _y_, one
pair per line, whitespace separated) and writes the signed area of the
polygon they describe, computed with the shoelace formula. Vertices given
in counter-clockwise order yield a positive area; clockwise order yields a
negative area. The polygon is implicitly closed, so the final vertex need
not repeat the first.

Lines that contain no leading number (blank lines, comments, color records,
etc.) are silently skipped. A line containing exactly one number is treated
as malformed input and aborts the program.

By default coordinates are read from standard input and the area is written
to standard output, making *cad_parea* convenient in a pipeline.

[[options]]
== OPTIONS

*-i* _input_::
  Read coordinates from _input_ instead of standard input. A value of
  *-* means standard input. May be given only once.

*-o* _output_::
  Write the computed area to _output_ instead of standard output. A value
  of *-* means standard output. May be given only once.

*-h*, *-?*::
  Print a usage message and exit.

[[examples]]
== EXAMPLES

.Area of a unit square from a file
====
[source,bash]
----
$ cat square.txt
0 0
1 0
1 1
0 1
$ cad_parea -i square.txt
1
----
The four vertices are listed in counter-clockwise order, so *cad_parea*
reports the positive area 1. Listing them clockwise would instead report -1.
====

.Area of a region traced interactively in a pipeline
====
[source,bash]
----
$ printf '0 0\n4 0\n4 3\n0 3\n' | cad_parea -o area.txt
$ cat area.txt
12
----
Coordinates are streamed in on standard input and the resulting area of the
4x3 rectangle (12) is written to the file _area.txt_, demonstrating how
*cad_parea* slots into a pipeline that generates vertex lists on the fly.
====

[[hints]]
== HINTS

A hole can be accommodated as a clockwise section connected to the periphery
by a line segment traced in both directions. Disjoint pieces may likewise be
connected by a line segment traced in both directions.

[[diagnostics]]
== DIAGNOSTICS

A line with a single numeric field is reported as bad input and *cad_parea*
exits with status 2. A degenerate polygon yields an area of 0.
