== NAME
fbcmap - load a color map onto a framebuffer

== SYNOPSIS

[source]
----
fbcmap [-h] [-F framebuffer] [-s squarescrsize] [-w scr_width] [-n scr_height] [-S squarescrsize] [-W scr_width] [-N scr_height] [map_number]
----

== DESCRIPTION

*fbcmap* writes a built-in color map to a framebuffer.
When invoked with no _map_number_ argument, or with a _map_number_ of
0, the standard linear (identity) color map is loaded.
Other map numbers select alternative mappings described below.

== OPTIONS

*-h*
Print help and exit.

*-F* _framebuffer_
Specifies the framebuffer device.
If not given, the *FB_FILE* environment variable is used
(see *brlcad*(1)).
If *FB_FILE* is not set, the default device for the system is used.

*-s* _squarescrsize_
Sets both the width and height of the framebuffer to
_squarescrsize_ pixels.

*-S* _squarescrsize_
Same as *-s*.

*-w* _scr_width_
Sets the framebuffer display width to _scr_width_ pixels.

*-W* _scr_width_
Same as *-w*.

*-n* _scr_height_
Sets the framebuffer display height to _scr_height_ pixels.

*-N* _scr_height_
Same as *-n*.

If no size options are given, the requested framebuffer size will be
the default for the selected device (typically 512×512 pixels).

An optional _map_number_ argument selects the color map to load.
The following maps are available:

0::
Linear (identity) color map (default).

1::
Reverse-linear (negative) map.

2::
Corrected for Polaroid 809/891 print film.

3::
Linear map with the first 100 entries set to black.

4::
Linear map with values 1–191 shifted up by 64 to boost dim
pictures.  Values 192 and above are full intensity.
Zero remains black.

5::
Gamma-correcting map from the University of Utah Alpha_1 project.
For general gamma correction, *fbgamma*(1) is preferred.

6::
Color deltas map.  Mostly black, with full-intensity colors at
multiples of 32: white (0), magenta (32), red (64), yellow (96),
green (128), cyan (160), blue (192).
Useful with *fbcmrot*(1) to visualize intensity relationships
across color planes.

8::
Ikcmap 8 false-color map.

9::
Ikcmap 9 false-color map.

10::
Solid black.

11::
Solid white.

12::
18% neutral grey.

Maps 10–12 are useful for adjusting a camera to a uniform field
without losing image memory.

== EXAMPLES

Load the standard linear color map onto the default framebuffer:

....
fbcmap
....

Load the reverse-linear (negative) map onto a specific framebuffer:

....
fbcmap -F /dev/fb0 1
....

Set the framebuffer to solid 18% grey at 1024×1024 resolution:

....
fbcmap -s 1024 12
....
