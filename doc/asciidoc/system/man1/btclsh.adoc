== NAME
btclsh - Tcl shell with BRL-CAD libraries loaded

== SYNOPSIS

[source]
----
btclsh [script [arg ...]]
----


[[description]]
== DESCRIPTION

*btclsh* is a Tcl interpreter with the BRL-CAD Tcl/Tk libraries
preloaded, giving Tcl scripts direct access to BRL-CAD's geometry and
database commands. It is the non-graphical counterpart to *bwish*(1):
it requires no display server and is suited to text-based interactive
work, scripting, and automation.

If a _script_ file is given and its name does not begin with a hyphen
(`-`), *btclsh* evaluates that file and exits. Any remaining _arg_
values are made available to the script through the Tcl variables
*argv* (the argument list), *argc* (the argument count), and *argv0*
(the script name).

With no _script_, or when the first argument begins with a hyphen,
*btclsh* runs interactively, reading commands from standard input. The
interactive session offers command-line editing (Emacs-style key
bindings, arrow-key history recall) and sources a startup file,
*~/.btclshrc* (*~/.btclshrc.tcl* on Windows), if present.

Besides the full Tcl command set, *btclsh* registers a few built-in
commands:

*history* [*-delays*] [*-outfile* _file_]::
  List the command history. With *-delays*, also report the elapsed
  time between commands; with *-outfile*, append the listing to _file_
  instead of printing it.

*hist* *add* _command_ | *next* | *prev*::
  Maintain the command history: add a command, or step to the next or
  previous entry (used internally by history recall).

*exit* [_status_] / *q* [_status_]::
  Leave the shell, returning the optional integer _status_ (default 0).

Because *btclsh* takes no command-line options, it has no OPTIONS
section.


[[examples]]
== EXAMPLES

.Scripting a geometry query with arguments
====
[source,bash]
----
cat > volume.tcl <<'EOF'
opendb [lindex $argv 0] y
puts [bb -v [lindex $argv 1]]
EOF

btclsh volume.tcl moss.g all.g
----
Runs a Tcl script that opens the geometry database *moss.g* (passed as
the first argument) and prints the bounding-box and volume report for
the *all.g* object using the libged *bb* command. This shows how
*btclsh* drives BRL-CAD database commands non-interactively, with
arguments forwarded through the Tcl *argv* variable.
====

.Interactive session
====
[source,bash]
----
btclsh
btclsh> opendb moss.g y
btclsh> ls
btclsh> q
----
Starts the interactive shell, opens a database, lists its objects with
the libged *ls* command, then quits.
====
