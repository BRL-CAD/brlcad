== NAME

g-iges - convert a BRL-CAD database to an IGES file

== SYNOPSIS

[source]
----
g-iges [-f|-t|-m] [-s] [-v] [-a _abs_tol_] [-r _rel_tol_] [-n _norm_tol_] [-d _dist_tol_] [-P _ncpu_] [-x _rt_debug_] [-X _nmg_debug_] [-o _output_file_] _database.g_ _object_ [_object_ ...]
----

[[description]]
== DESCRIPTION

*g-iges* converts the specified _object_(s) from a BRL-CAD _database.g_ database
into an IGES version 5.1 file.

By default a CSG IGES file is produced: every region is written as a Boolean
tree, with each solid represented by its IGES CSG equivalent. Any solid that has
no IGES CSG equivalent is tessellated (using the tessellation tolerances) and
written as a boundary-representation (B-Rep) object. The *-f*, *-t*, and *-m*
options select alternative output modes; they are mutually exclusive, and the
last one given wins.

If no output file is given with *-o*, the IGES file is written to standard
output. The inverse translation, from IGES to BRL-CAD, is performed by
*iges-g*(1).

[[options]]
== OPTIONS

*-f*::
  Facetized B-Rep output. Every region is tessellated, its Boolean operations
  are evaluated, and the result is written as a single manifold-solid B-Rep
  object. Tessellation accuracy is governed by *-a*, *-r*, and *-n*.

*-t*::
  Trimmed-surface output (experimental). Each region is written as a collection
  of trimmed NURBS derived from the facets of the tessellated, Boolean-evaluated
  region.

*-m*::
  Multi-file output: write a separate IGES file for each converted region.
  Implies *-t*, and the *-o* argument must name an existing directory in which
  the per-region files are created.

*-s*::
  Write the faces of B-Rep objects as planar NURBS surfaces.

*-a* _abs_tol_::
  Absolute tessellation tolerance, in millimeters. Default 0.0 (disabled).

*-r* _rel_tol_::
  Relative tessellation tolerance, as a fraction of object size. Default 0.01.

*-n* _norm_tol_::
  Surface-normal tessellation tolerance. Default 0.0 (disabled).

*-d* _dist_tol_::
  Minimum distance between distinct points, in millimeters. Default 0.0005.

*-P* _ncpu_::
  Number of processors to use during tessellation. Default 1.

*-v*::
  Verbose output; repeat for more detail.

*-x* _rt_debug_::
  Set the *librt* raytrace debug flag (hexadecimal). See raytrace.h for values.

*-X* _nmg_debug_::
  Set the NMG debug flag (hexadecimal). See nmg.h for values.

*-o* _output_file_::
  Write the IGES output to _output_file_ instead of standard output. With *-m*,
  this argument must be a directory.

[[examples]]
== EXAMPLES

.Convert selected regions to a CSG IGES file
====
[source,bash]
----
g-iges -o assembly.iges assembly.g hull turret
----
Writes the hull and turret regions of assembly.g as a CSG IGES file, preserving
the Boolean tree where IGES primitive equivalents exist and tessellating the
rest.
====

.Produce a faceted B-Rep IGES file with tight tolerances
====
[source,bash]
----
g-iges -f -a 0.1 -n 0.01 -o widget.iges widget.g widget
----
Tessellates the widget region to within an absolute tolerance of 0.1 mm and a
surface-normal tolerance of 0.01 radians, then writes it as a manifold-solid
B-Rep, suitable for downstream CAD packages that prefer evaluated geometry over
CSG.
====

.Split a model into one IGES file per region
====
[source,bash]
----
mkdir parts
g-iges -m -o parts model.g topgroup
----
Writes each region under topgroup as its own trimmed-NURBS IGES file inside the
parts directory, which is convenient for importing assemblies part-by-part.
====
