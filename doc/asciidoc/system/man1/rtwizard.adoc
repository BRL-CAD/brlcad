== NAME
rtwizard - composite BRL-CAD images from color, ghost, and edge-line renderings

== SYNOPSIS

[source]
----
rtwizard [options] [-i] _geometry.g_ [-o] _output.{pix|png}_ [object ...]
----


[[description]]
== DESCRIPTION

*rtwizard* orchestrates several BRL-CAD utilities to produce composite
images of geometry in prescribed styles.  Full-color ray tracings are
produced by *rt*(1), "hidden-line" edge drawings by *rtedge*(1), and
semi-transparent greyscale "ghost" renderings by *rt*(1) combined with
BRL-CAD image-manipulation tools.  These layers are composited into a
single image written to a *pix*(5) or PNG file, or displayed on a
framebuffer.

*rtwizard* has two modes.  With no arguments (or with *--gui*) it launches
a wizard-style graphical interface that guides the user through building an
image interactively.  Given sufficient options on the command line it runs
non-interactively, which is useful for scripting and bulk rendering.  If a
command line invocation lacks enough information to produce the requested
image, *rtwizard* falls back to the GUI unless *--no-gui* is given.

The kind of image produced is governed by the *--type* option, or inferred
from which object lists are supplied.  The six picture types are:

.RtWizard Image Types
[cols="3*"]
|===
|Type |Name |Description

|A |Simple Full-Color Image |Standard *rt* image.
|B |Simple Line Drawing |Standard *rtedge* image.
|C |Highlighted Image |Full color *rt* image enhanced with *rtedge* lines.
|D |Mixed Full Color and Edges |Like C, but objects are selectively edge-enhanced.
|E |Ghost Image with Inserts |Full color elements over faded greyscale context.
|F |Ghost Image with Inserts and Edges |A Type E image further enhanced with *rtedge* lines.
|===

For compatibility with *rt*(1), the first unflagged argument naming an
existing .g file is taken as the input geometry, and the first unflagged
argument with a supported image extension (.pix, .png, .ppm, .bw, .dpix) is
taken as the output file.  Any remaining unflagged arguments are treated as
color objects.  Because this can be ambiguous, the *-i* and *-o* flags are
preferred.

[[options]]
== OPTIONS

*-h*, *--help*::
  Print the options summary and exit.

*--help-dev*::
  Print the developer/programmatic options and exit.

*-i* _filename_, *--input-file* _filename_::
  Input .g geometry database.  Overrides any unflagged .g argument.

*-o* _filename_, *--output-file* _filename_::
  Output image file.  A .png extension produces a PNG image; otherwise
  a BRL-CAD *pix*(5) image is written.  Other supported extensions are
  .ppm, .bw, and .dpix.

*-s* _#_, *--size* _#_::
  Width and height of a square output image, in pixels (default 512).
  Overridden per-axis by *-w* and *-n*.

*-w* _#_, *--width* _#_::
  Output image width in pixels (overrides the width from *-s*).

*-n* _#_, *--height* _#_::
  Output image height in pixels (overrides the height from *-s*).

*-t* _A|B|C|D|E|F_, *--type* _A|B|C|D|E|F_::
  Picture type to produce (see the table above).  If a type is requested
  but the supplied object lists are insufficient for it, *rtwizard* exits
  with an error.

*-c* _obj[,...]_, *--color-objects* _obj[,...]_::
  Comma-separated list of objects to render in full color.  Quote the list
  if names contain spaces (e.g. -c "obj1,obj 2").

*-l* _obj[,...]_, *--line-objects* _obj[,...]_::
  Comma-separated list of objects to render as edge lines.

*-g* _obj[,...]_, *--ghost-objects* _obj[,...]_::
  Comma-separated list of objects to render as ghosted (faded) context.

*-a* _angle_, *--azimuth* _angle_::
  Set the azimuth view angle, in degrees.

*-e* _angle_, *--elevation* _angle_::
  Set the elevation view angle, in degrees.

*--twist* _angle_::
  Set the twist view angle, in degrees.

*-z* _value_, *--zoom* _value_::
  Zoom factor as a multiplier on the default view size.  Values greater
  than one enlarge the object; values less than one shrink it.

*--center* _x/y/z_::
  Set the xyz coordinates of the view center.

*--eye_pt* _x/y/z_::
  Set the eye point.  Part of a complete low-level view specification
  (with *--viewsize* and *--orientation*); a complete low-level
  specification overrides the high-level view options.

*--viewsize* _value_::
  Set the view size directly (low-level view specification).

*--orientation* _x/y/z/w_::
  Set the view orientation quaternion (low-level view specification).

*-P* _angle_, *--perspective* _angle_::
  Set the perspective angle, in degrees.

*-C* _color_, *--background-color* _color_::
  Background color for full-color ray tracing, as R/G/B or hex (default
  white).

*--line-color* _color_::
  Color for edge lines, as R/G/B or hex.  The keyword "region" uses each
  region's own color for its lines.

*--non-line-color* _color_::
  Color used for the non-line areas of edge renderings.

*-G* _value_, *--ghost-intensity* _value_::
  Visibility of ghosted objects (default 6.0).

*-O* _#_, *--occlusion* _#_::
  Occlusion mode (1 to 3) for line rendering (default 1).  See *rtedge*(1).

*--gui*::
  Force the graphical interface even when the command line specifies enough
  information to produce an image.

*--no-gui*::
  Force command-line operation even when the supplied information is
  insufficient.  Overridden by *--gui* if both are given.

*-d* _device_, *--fbserv-device* _device_::
  Framebuffer device for *fbserv*(1) to use instead of the default
  memory-only interface (e.g. -d /dev/wgl).  See *fbhelp*(1) for available
  devices.  A framebuffer started by *rtwizard* is closed automatically
  after image generation.

*-p* _#_, *--fbserv-port* _#_::
  TCP port for the *fbserv*(1) framebuffer.  An existing framebuffer server
  may be reused; in that case *rtwizard* leaves it open after rendering.

*--benchmark*::
  Benchmark mode: disable randomness for reproducible timing.

*--cpu-count* _#_::
  Number of CPUs to use.

*--pid-file* _filename_::
  File used for tracking process IDs.

*--log-file* _filename_::
  Write debugging output to this file.

*-v* _#_, *--verbose* _#_::
  Set the verbosity level.

[[examples]]
== EXAMPLES

.Highlighted color-plus-edge image at a custom view
====
[source,bash]
----
rtwizard -i m35.g -c component/power.train -g component -l component \
         -a -35 -e 15 -z 1.6 -o m35.png
----
Render the m35 sample model with its power train highlighted in full color
over a ghosted, edge-lined truck body, viewed from azimuth -35 and
elevation 15 and zoomed in 1.6x, saving the result as a PNG image.
====

.Bulk rendering on the command line
====
[source,bash]
----
for g in *.g; do
    rtwizard --no-gui -t A -i "$g" -c all -s 1024 -o "${g%.g}.png"
done
----
Non-interactively produce a 1024x1024 full-color image of the "all" object
in every .g file in the current directory, naming each output after its
database.  *--no-gui* ensures the script never blocks on the GUI.
====

.Live preview on a framebuffer
====
[source,bash]
----
rtwizard -d /dev/wgl -i m35.g -l component
----
Display a line drawing of the m35 truck in an OpenGL framebuffer window
rather than writing it to a file.
====
