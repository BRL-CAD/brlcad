== NAME
bot_dump - export polygonal BoT primitives without performing evaluation

== SYNOPSIS

[source]
----
bot_dump [-b] [-n] [-F] [-m directory] [-o file] [-t dxf | glb | gltf | obj | sat | stl] [-u units] [--displayed] [--viewdata] [--materials | --no-materials] geom.g [bot1 bot2 ...]
----


[[description]]
== DESCRIPTION

*bot_dump* exports the raw definition of polygonal Bag of Triangles (BoT) primitives from a BRL-CAD database to various standard formats. This operation does not evaluate Boolean combinations, ensuring that the exported geometry matches the primitive's exact original representation. 

The *-b* option specifies that the output file should use a binary format, which is applicable to formats supporting both ASCII and binary variants, such as STL and glTF (as GLB).

The *-n* or *--normals* option includes surface normals in the output. This is only supported for certain formats like OBJ. Normals are natively required in STL and are not supported in DXF or SAT.

The *-F* or *--full-precision* option writes floating-point values at maximum precision. Note that this is non-standard for some formats like glTF.

The *-m* _directory_ or *--output-directory* _directory_ option outputs each dumped BoT into a separate file within the specified directory. This cannot be used simultaneously with *-o*.

The *-o* _file_ or *--output-file* _file_ option specifies a single output file for all exported data. If neither *-o* nor *-m* is specified, the output is printed to standard output (except for binary formats).

The *-t* _type_ option specifies the output file format. Supported formats are *dxf*, *glb*, *gltf*, *obj*, *sat*, and *stl*. If no format is specified but an output file is given with an extension, the format is inferred. Otherwise, the default format is *stl*.

The *-u* _units_ option converts coordinates from internal millimeters to the specified unit (e.g., "in", "cm", "m") during export.

The *--displayed* option exports currently displayed geometry (useful when invoked from within MGED).

The *--viewdata* option exports non-geometry view-specific data (e.g., object colors).

The *--materials* option exports resolved color and material information when the format supports it. This is enabled by default for supported formats.

The *--no-materials* option disables color and material export, preserving geometry-only output.


[[examples]]
== EXAMPLES

Export a single BoT to a DXF file:

[source]
----
$ bot_dump -o some_bot_dump.dxf -t dxf sample.g some_bot
----

Export multiple BoTs as binary glTF (GLB) to a directory:

[source]
----
$ bot_dump -m output_dir -t glb sample.g bot1 bot2 bot3
----

Export BoTs to OBJ with normals and default color/material export:

[source]
----
$ bot_dump -n --materials -o output.obj -t obj sample.g my_bot
----
