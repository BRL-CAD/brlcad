== NAME
g-obj - convert BRL-CAD geometry to a Wavefront OBJ file

== SYNOPSIS

[source]
----
g-obj [-h|-?] [-m] [-v] [-i] [-u] [-x rt_debug] [-X nmg_debug]
      [-a abs_tol] [-r rel_tol] [-n norm_tol] [-D dist_tol]
      [-P ncpu] [-e error_file] [-o output.obj] database.g object(s)
----


[[description]]
== DESCRIPTION

*g-obj* converts the specified _object(s)_ from a BRL-CAD _database.g_
file to the Wavefront _OBJ_ format. Each region is tessellated into an
NMG, evaluated, and triangulated, so the resulting OBJ contains only
triangular faces.

OBJ output is written to _output.obj_ when the *-o* option is supplied;
otherwise it is written to standard output. Coordinates are emitted in
millimeters by default, or in inches with *-i*.

When writing to a named OBJ file, *g-obj* also writes a companion _.mtl_
material file with the same base name. The material file preserves
effective BRL-CAD colors (colors set on geometry, inherited from
combinations, or supplied by region color-id mappings). If OBJ output
goes to standard output no companion _.mtl_ file can be created, and
BRL-CAD colors are therefore not exported.

See *g-stl*(1) and *g-ply*(1) for related polygonal exporters.


[[options]]
== OPTIONS

*-h*, *-?*::
  Print a usage summary and exit.

*-i*::
  Write output coordinates in inches instead of the default millimeters.

*-m*::
  Emit `usemtl` statements. When a companion _.mtl_ file is written, the
  generated material names preserve the BRL-CAD GIFT material id, LOS, and
  aircode, with a color suffix appended when needed to distinguish
  different RGB assignments.

*-u*::
  Write vertex normals (`vn`) and reference them from the face records.

*-v*::
  Verbose output, reporting progress for each region processed.

*-a* _abs_tol_::
  Set the absolute tessellation tolerance, the maximum allowable distance
  (in millimeters) between a tessellated face and the true surface.
  Setting this clears the relative tolerance.

*-r* _rel_tol_::
  Set the relative tessellation tolerance as a fraction of the bounding
  box size. The default is 0.01.

*-n* _norm_tol_::
  Set the surface-normal tessellation tolerance, the maximum allowable
  angle (in degrees) between adjacent face normals. Setting this clears
  the relative tolerance.

*-D* _dist_tol_::
  Set the calculational distance tolerance, in millimeters. Vertices
  closer together than this distance are treated as the same vertex.

*-P* _ncpu_::
  Request use of _ncpu_ processors during processing.

*-e* _error_file_::
  Write diagnostics to _error_file_. By default diagnostics go to standard
  error.

*-o* _output.obj_::
  Write OBJ output to _output.obj_, and a companion _.mtl_ file with the
  same base name.

*-x* _rt_debug_::
  Set the librt debug flags to the given hexadecimal value.

*-X* _nmg_debug_::
  Set the libnmg debug flags to the given hexadecimal value.


[[examples]]
== EXAMPLES

.Export a region to OBJ with materials
====
[source,bash]
----
g-obj -o havoc.obj havoc.g havoc
----
Tessellate the _havoc_ region from _havoc.g_ and write _havoc.obj_ along
with a companion _havoc.mtl_ that captures the model's BRL-CAD colors.
====

.Export with vertex normals and material codes
====
[source,bash]
----
g-obj -m -u -a 0.5 -o tank.obj tank.g turret hull
----
Convert the _turret_ and _hull_ objects with a 0.5 mm absolute
tessellation tolerance, emitting vertex normals and `usemtl` statements
whose names encode GIFT material, LOS, and aircode values.
====

.Pipe OBJ output in inches to standard output
====
[source,bash]
----
g-obj -i sample.g sample_object > sample.obj
----
Write the geometry in inches to standard output, redirected into
_sample.obj_. Because no named OBJ file is opened, no companion _.mtl_
file is produced and colors are not exported.
====
