== NAME
pixembed - embed a pix image in a larger frame, replicating border pixels

== SYNOPSIS

[source]
----
pixembed [-b border_inset] [-s squareinsize] [-w inwidth] [-n inheight]
         [-S squareoutsize] [-W outwidth] [-N outheight] [in.pix] > out.pix
----


[[description]]
== DESCRIPTION

*pixembed* reads a 3-byte/pixel RGB *pix*(5) image and centers it within a
larger output *pix*(5) image. Rather than padding the surrounding border with
black, *pixembed* replicates the boundary pixels of the input outward to fill
the entire margin.

Because the extreme edge pixels of an image are sometimes noisy or degraded,
the _border_inset_ may be increased so the replicated color is sampled from a
row or column slightly inside the actual edge of the input.

If no _in.pix_ is given, the image is read from standard input; the embedded
result is always written to standard output. Both input and output dimensions
default to 512x512. The output size must be greater than or equal to the input
size, and the border inset must be smaller than the input width.

[[options]]
== OPTIONS

*-b* _border_inset_::
  Inset, in pixels, from the actual image edge at which the replicated border
  color is sampled. The default is 0 (the exact edge pixels are replicated).

*-s* _squareinsize_::
  Set both the width and height of the input image to _squareinsize_ pixels.

*-w* _inwidth_::
  Set the width of the input image to _inwidth_ pixels (default 512).

*-n* _inheight_::
  Set the height of the input image to _inheight_ pixels (default 512).

*-S* _squareoutsize_::
  Set both the width and height of the output image to _squareoutsize_ pixels.

*-W* _outwidth_::
  Set the width of the output image to _outwidth_ pixels (default 512).

*-N* _outheight_::
  Set the height of the output image to _outheight_ pixels (default 512).

[[examples]]
== EXAMPLES

.Embed a 512x512 image into a 1024x1024 frame
====
[source,bash]
----
pixembed -s 512 -S 1024 < input.pix > output.pix
----
Centers the 512x512 input within a 1024x1024 image, replicating the input's
edge pixels to fill the surrounding border, then writes the larger image to
output.pix.
====

.Generate a clean matte for compositing
====
[source,bash]
----
pixembed -w 640 -n 480 -W 800 -N 600 -b 4 render.pix | pix-png -w 800 -n 600 > matte.png
----
Embeds a 640x480 render into an 800x600 frame, sampling the border color from
4 pixels inside the edge to avoid noisy outer rows, and pipes the result
straight into *pix-png*(1) to produce a PNG ready for compositing.
====
