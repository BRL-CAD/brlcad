== NAME
libdm - display manager library for BRL-CAD

== SYNOPSIS

[source,c]
----
#include <dm.h>
----

Link with *-ldm*.

The public interface is organized across the headers included by
_dm.h_, namely _dm/defines.h_, _dm/fbserv.h_, _dm/util.h_, and
_dm/view.h_.

== DESCRIPTION

LIBDM provides BRL-CAD's display manager API. A display manager can be
thought of as a configurable drawing machine: it consumes generic,
device-independent drawing commands -- primarily vector lists (vlists)
that describe points, lines, and polygons in three dimensions, along
with two-dimensional points, lines, and text -- and translates them into
the drawing operations of a specific output device or window system.

The library abstracts the underlying rendering target behind a common
_struct dm_ interface, so that applications can draw the same geometry to
any supported backend without change. Backends provided with BRL-CAD
include X11, OpenGL (via GLX on X11 and WGL on Windows), a Qt/OpenGL
target, a software rasterizer, PostScript output, UNIX plot output, a
text backend, and a null device. LIBDM also integrates with the BRL-CAD
framebuffer facilities, including the framebuffer server support in
_dm/fbserv.h_.

The complete API, including data structures and function signatures, is
documented in the headers noted above and in the Doxygen-generated
reference for the LIBDM module.
