== NAME
regions - write a region and Boolean member summary table to a file

== SYNOPSIS

[source]
----
regions file object(s)
----


[[description]]
== DESCRIPTION

The *regions* command traverses the hierarchy of one or more `objects` and writes a summary of all regions found to `file`. The file is opened in write mode and overwritten if it already exists.

*regions* is one of four related commands that all share the same underlying logic but produce different levels of detail:

[cols="1,4"]
|===
|Command |Output detail

|`idents`
|Region-level summary only: region count, ident, air code, material code, LOS, and path. No primitives listed.

|`regions`
|Region-level summary with the Boolean formula (member names and operators) for each region.

|`solids`
|Full summary down to primitives: same as `regions`, plus the evaluated parameters of each primitive shape with accumulated transformation matrices applied.

|`tables`
|Not a valid standalone command. Invoking `tables` directly returns an input error. Use one of the three specific commands above.

|===

=== Output format

The output file is a fixed-width text table. Each region entry includes the following fields in order:

. *Region count* — sequential index of the region as encountered.
. *Region ID* (`ident`) — the region's identifier number.
. *Air code* — air region code (0 if not an air region).
. *Material code* (`GIFTmater`) — GIFT material code.
. *LOS* — line-of-sight percentage (0–100).
. *Path* — the full path from the listed object to the region.
. *Boolean members* — the list of member primitives and Boolean operators (`u`, `-`, `+`) forming the region. Combination sub-members are shown as `RG` entries; unknown object types as `??`.

A header is written at the top of the file containing the command name, database filename, username, database title, units, and the list of objects processed.

After the table, a summary line is appended: `Number Primitives = N  Number Regions = N`.

=== Primitive numbering

Each primitive in the Boolean formula is assigned a unique solid number in brackets (e.g. `[3]`). The same primitive used in multiple regions is assigned the same number if the accumulated transformation matrix is identical; a different transformation produces a different number. This uniqueness is tracked via a temporary file at `/tmp/mged_discr`.

[[examples]]
== EXAMPLES

.Write a regions summary for two groups
[example]
====
mged> **regions regions_file group1 group2**

Writes a region and Boolean member summary for all regions reachable from `group1` and `group2` to `regions_file`. The result string reports how many primitives and regions were processed.
====

.Write a summary for a specific top-level object
[example]
====
mged> **regions hull_report.txt vehicle.c**

Writes the regions table for all regions under `vehicle.c` to `hull_report.txt`.
====
