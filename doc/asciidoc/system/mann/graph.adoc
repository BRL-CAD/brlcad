== NAME

graph - generate a database hierarchy graph

== SYNOPSIS

[source]
----
graph [options] output.svg
graph --igraph [show|positions]
----

== DESCRIPTION

The *graph* command creates a diagram of the currently open database's
hierarchy.  Its normal form writes an SVG document to _output.svg_.  The
output filename must use the `.svg` suffix.

The `--igraph` form retains the textual graph operations used by older
GraphEditor/igraph scripts.  It takes exactly one operation: `show` reports
nodes and connections, and `positions` reports object-node positions.

The command requires the Adaptagrams support library.  Builds without that
library register the command but report that graph generation is unavailable.

== OPTIONS

*-h*, *--help*::
Print command usage.

*--tree*::
Collapse the diagram to a spanning tree, limiting each node to at most one
incoming graph edge.

*--skip-routing*::
Write nodes without calculating connector routes.

*--depth* _depth_::
Limit hierarchy traversal to a non-negative depth.

*--root-obj* _object_::
Use _object_ as a traversal root.  This option may be repeated to draw several
rooted subgraphs.

*--igraph*::
Select the legacy textual mode.  Follow it with `show` or `positions` rather
than an SVG filename.

== EXAMPLES

.Write an SVG of the complete hierarchy
[example]
====
mged> *graph hierarchy.svg*
====

.Write a depth-limited graph rooted at an assembly
[example]
====
mged> *graph --root-obj all.g --depth 3 assembly.svg*
====

.Use the legacy textual position report
[example]
====
mged> *graph --igraph positions*
====
