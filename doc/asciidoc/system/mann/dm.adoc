== NAME

dm - Interact with display-manager instances.

== SYNOPSIS

[source]
----
dm [-h] [-v] subcommand [args]
dm attach [--dm name] [--view name] type [name]
dm bg [color [color]]
dm debug [level]
dm get [--dm name] [variable ...]
dm set [--dm name] [variable [value]]
dm {width | height | type} [name]
dm {types | list | initmsg}
----

[[description]]
== DESCRIPTION

The *dm* command interacts with display-manager instances at a lower level.
Top-level options must precede the subcommand. The `-v`/`--verbose` option may
be repeated to increase report detail.

The *attach* subcommand creates a display manager of the specified _type_.
Its optional positional _name_ and `--dm name` name the new manager; supplying
both with different values is an error. Without a name, a unique name based on
the manager type is generated. `--view name` associates the manager with an
existing view. Otherwise it uses the current view when available, or creates a
view for the manager.

The *bg* subcommand queries the current background without arguments. One
color sets a solid background; two colors set the two gradient endpoints.
Colors accept common BRL-CAD forms, including `r/g/b`, `r,g,b`, `r;g;b`, and
three separate components.

The *debug* subcommand queries or sets the display manager's integer debug
level. *type*, *width*, and *height* query the selected manager; their
optional _name_ selects a manager other than the current one. *types* lists
the display manager types supported by the current build, *list* lists known
instances, and *initmsg* prints display-manager plug-in initialization
messages.

The *get* and *set* subcommands query display-manager variables. `dm get`
and `dm set` with no variable print all variables. `dm set variable` queries
one variable, while `dm set variable value` assigns it. `dm get variable ...`
queries each requested variable. `--dm name` selects a manager for either
subcommand.

[[options]]
== OPTIONS

*-h*, *--help*::
Print native command-tree help. When followed by a subcommand, print help for
that child.

*-v*, *--verbose*::
Increase top-level report detail. This option may be repeated.

[[examples]]
== EXAMPLES

.List the display-manager types available in this build.
[example]
====
mged>**dm types**

Lists available types.

====

.List the current display manager's variables.
[example]
====
mged>**dm set**

Prints its internal variables.

====

.Set a display-manager variable.
[example]
====
mged>**dm set perspective 1**

Sets `perspective` when that variable is supported.

====

.Set a two-color background gradient.
[example]
====
mged>**dm bg 0/0/32 32/32/96**

Sets the two background endpoints.

====
