:doctype: manpage
:mansource: BRL-CAD
:manmanual: BRL-CAD User Commands

== NAME
saveview - Write the current MGED view and displayed objects to an executable raytrace script.

== SYNOPSIS

[source]
----
saveview [-e command] [-i input.g] [-l logfile] [-o output] filename [args]
----


[[description]]
== DESCRIPTION

The *saveview* command writes a shell script describing the current MGED view and the objects currently displayed. The generated script runs a command in *-M* mode, feeding it the saved viewing parameters on standard input.

By default, the generated script runs *rt -M*. If perspective is enabled in the current view, the script also adds the matching *-p* option. Any additional _args_ supplied after _filename_ are copied into the generated command line unchanged.

The script is opened in append mode. If _filename_ already exists, the new script block is appended to the existing file. The file is also marked executable.

Unless overridden, the script:

* writes image output to a file derived from _filename_ with `.pix` appended;
* writes stderr output to a file derived from _filename_ with `.log` appended;
* uses the currently opened database file as the geometry input;
* includes the currently displayed objects as the raytrace object list.

When deriving default `.pix` and `.log` names, a trailing `.sh` suffix is removed from _filename_ first, if present. For example, `saveview render.sh` defaults to `render.pix` and `render.log`, while `saveview render.rt` defaults to `render.rt.pix` and `render.rt.log`.

The following options are supported:

* *-e* _command_ - Run _command_ instead of `rt`. The command is invoked with `-M`. If _command_ begins with `nirt`, *saveview* does not emit a default `-o` output option.
* *-i* _input.g_ - Use _input.g_ in the generated script instead of the currently opened database pathname.
* *-l* _logfile_ - Write stderr redirection to _logfile_ instead of the default derived `.log` file.
* *-o* _output_ - Pass `-o output` to the generated command instead of the default derived `.pix` file.

The resulting script contains the view state needed to reproduce the current display, including view size, orientation, eye point, and the displayed object list. Scripts generated by *saveview* can be read back into MGED with *loadview* to restore the same view and displayed objects.


[[examples]]
== EXAMPLES

These examples show how *saveview* records the current MGED view into a reusable shell script.

.Write a default raytrace script for the current view.
[example]
====
mged>**saveview render.rt**

Appends an executable shell script to _render.rt_ that runs `rt -M` on the currently displayed objects using the current view.

====

.Write a script that passes an explicit image size to the generated raytrace command.
[example]
====
mged>**saveview render.rt -s1024**

Appends a script to _render.rt_ and passes `-s1024` through to the generated raytrace command.

====

.Write a script that runs `nirt` instead of `rt`.
[example]
====
mged>**saveview -e nirt raycast.sh**

Appends an executable script to _raycast.sh_ that runs `nirt -M` using the current MGED view.

====
