== NAME
pset - set a scalar parameter of a primitive

== SYNOPSIS

[source]
----
pset obj attribute val
----

[[description]]
== DESCRIPTION

The *pset* command sets a scalar parameter of the BRL-CAD primitive `obj` to the given value `val`. The parameter is selected by a single-letter `attribute` code. The modified object is written back to the database.

The `val` argument must be a single floating-point number. Path prefixes on `obj` (e.g. `comb/part.s`) are accepted; the trailing component name is used for the lookup.

Currently only METABALL primitives are supported. Other primitive types return an error.

=== METABALL attributes

[cols="1,3"]
|===
|Attribute |Description

|`m` or `M`
|Render method. Valid values are 0 (Metaball), 1 (iso-surface), and 2 (Blob). Values below 0 are clamped to 0; values above 2 are clamped to 2.

|`t` or `T`
|Threshold value. Sets the field threshold used during metaball rendering. If a negative value is given, its absolute value is used.

|===

[[examples]]
== EXAMPLES

.Set the render method of a metaball
[example]
====
mged> **pset blob.s m 2**

Sets the render method of the metaball primitive `blob.s` to Blob mode (method 2).
====

.Set the threshold of a metaball
[example]
====
mged> **pset cloud.s t 1.5**

Sets the field threshold of the metaball primitive `cloud.s` to 1.5.
====
