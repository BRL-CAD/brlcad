:doctype: manpage
:mansource: BRL-CAD

== NAME
view, view2 - get or set view parameters (local units).

== SYNOPSIS

[source]
----
view [*-h* | *--help*] [*-v* | *--verbose*] [*-V* _name_ | *--view* _name_] _subcommand_ [_args_]
view2 [*-h* | *--help*] [*-v* | *--verbose*] [*-V* _name_ | *--view* _name_] _subcommand_ [_args_]
----


[[description]]
== DESCRIPTION

Gets/sets view parameters in local units. The *view* and *view2* commands use
the same parser-owned command tree: top-level options must precede the
subcommand, after which the selected operation owns its arguments. This keeps
option completion and validation in the correct phase instead of mistaking an
operation argument for a top-level option.

The direct tree subcommands are *ae* (with accepted compatibility alias
*aet*), *center*, *dir*, *eye*, *faceplate*, *gobjs*, *height*, *independent*, *knob*,
*list*, *lod*, *obj* (with accepted compatibility alias *objs*), *quat*,
*selections*, *size*, *snap*, *vZ*, *width*, and *ypr*. Completion suggests
the canonical spellings *ae* and *obj*. A `--` marker ends top-level option
parsing before the subcommand.

The operation descriptions below include the established classic view forms
and the syntax implemented by their individual operation commands.


[[options]]
== OPTIONS

*-h*, *--help*::
Print help.

*-v*, *--verbose*::
Increase command verbosity.

*-V*, *--view* _name_::
Operate on the specified view instead of the current GED view.


[[subcommands]]
== SUBCOMMANDS


*ae* [[_-i_] _azimuth elevation_ [_twist_]]::
Get (without arguments) or set the azimuth, elevation, and (optional) twist. If only 2 arguments are supplied, a twist of 0 is assumed. The _-i_ flag can be used to set the AET incrementally. Setting the azimuth and elevation also changes the eye point while keeping the center and view size static.


*align* _[-i] x y z_::
Aligns the eye point with a desired point and the current center (eye -> pt -> center). The _-i_ flag can be used to align from the opposite direction (eye -> center -> pt). Align changes the eye point, azimuth, and elevation while keeping the center and view size static.


*auto* [_-s scale_] [_object ..._]::
Resets the view size, center, and eye point such that the framed objects are visible in the graphics window. With no arguments, all displayed objects are framed; if a list of _object_ names (or full paths) is supplied, the view is instead sized to frame only those objects (which need not be displayed). The _-s_ (or _--scale_) option sets the view scale (model scale relative to view size). Azimuth and elevation remain static.


*center* [_x y z_]::
Get (without arguments) or set the center. Setting the center also changes the eye point while keeping the azimuth, elevation, and view size static.


*dir* [[_-i_] _x y z_ [_twist_]]::
Get (without arguments) or set the view in the form of a direction vector and optional twist. Setting the direction also changes the azimuth, elevation, and eye point while keeping the center and view size static. The _-i_ flag can be used to invert all values (for both getting and setting). The direction vector that is returned (without arguments) does not include the twist; the twist can be acertained by taking the inverse of the twist returned with the *ae* subcommand.


*eye* [_x y z_]::
Get (without arguments) or set the eye point. Setting the eye point also changes the center while keeping the azimuth, elevation, and view size static.


*faceplate* [_options_] _subcommand_ [_args_]::
*Not Implemented.*
+
Manipulate options for the faceplate.


*lookat* _x y z_::
Adjusts the current view such that the eye is looking in the direction of the given coordinates, but does not move the eye point nor change the view size. Center, azimuth, and elevation will change.


*obj* _name_ *axes* [_options_] _subcommand_ [_args_]::
Create or edit a model-space data-axes view object.  The axes subcommands are
`create`, `pos`, `size`, `line_width`, and `axes_color`; see
*view_obj_axes*(nged) for their syntax and query/set behavior.

*obj* [*-L*] [*-G* | *--view_only*] [_name_ [_operation_ [_args_]]]::
List view objects when _name_ is omitted, or operate on the named view object.
`-L` limits lookup or creation to the current view; `-G` selects
geometry-backed objects and `--view_only` selects view-only objects.  The
operations `draw [UP|DOWN]`, `del`, `lcnt`, and `update [x y]` respectively
query/set drawing state, delete a view-only object, report vlist entities, and
refresh an object (with an optional nonnegative screen-coordinate pair).
`color [-r|--recursive] [color]` queries or sets the object color; a color may
be a packed color accepted by the common color parser or three separate RGB
components.  `arrow [0|1|width [size]|length [size]]` controls an arrow for
objects that support one.  `label`, `line`, and `polygon` select their
corresponding object-operation families.  `line` has native `create` and
`append` operations, each accepting a packed XYZ point or three finite
coordinates.  `label create text x y [z] [px py pz]` creates a label at a
screen or model point, with an optional target line.  `polygon` retains its
native `create`, point-editing, contour, fill, import/export, overlap, and
CSG operations; see *view_obj_polygon*(nged) for their syntax and validation
rules.


*print*::
Returns the current azimuth, elevation, and twist; direction vector; center; eye point; and view size.


*quat* [_v1 v2 v3 v4_]::
Get (without arguments) or set the view in the form of a quarternion. Setting the quarternion also changes the azimuth, elevation, and eye point while keeping the center and view size static.


*save* [_-e command_] [_-l scriptlogfile_] [_-o scriptoutputfile_] [_-i scriptinputfile_] _file_ [_args_]::
Saves the current viewing parameters in the specified _file_ in the form of a shell script that will run the BRL-CAD rt program. The _-e_ flag specifies a command for the script to run instead of the default rt. The _-l_ flag specifies a logfile for the script to write instead of the default inputfile.log. The _-o_ flag specifies the output filename for the script to create instead of the default inputfile.pix. The _-i_ flag specifies the input filename for the script to load instead of the current database (default). Various arguments can be added to the end, such as -s1024 (see the man page for the *rt* command).


*size* [__view_size__]::
Get (without arguments) or set the view size. Setting the view size also changes the eye point while keeping the azimuth, elevation, and center static.


*ypr* [_yaw pitch roll_]::
Get (without arguments) or set the view in the form of yaw, pitch, roll. A ypr of 0 0 0 is a rear view (AE = 180, 0). Setting the ypr also changes the azimuth, elevation, and eye point while keeping the center and view size static.




[[examples]]
== EXAMPLES

.Get the center.
[example]
====
mged>**view center**

Gets the center.

====

.Set the center at the origin of model space.
[example]
====
mged>**view center 0 0 0**

Sets the center at the origin of model space.

====

.Get the current yaw, pitch, and roll
[example]
====
mged>**view ypr**

Returns the yaw, pitch, and roll of the current view.

====

.Set the current yaw, pitch, and roll
[example]
====
mged>**view ypr 90 0 0**

Sets the view using yaw, pitch, and roll so that the ae is 270, 0 (right).

====

.Increase the azimuth by 10 degrees
[example]
====
mged>**view ae -i 10 0**

Sets the view by increasing the current azimuth by 10 degrees

====

.Save current viewing parameters in a specified file in the form of a shell script to run the BRL-CAD rtedge program
[example]
====
mged>**view save -e rtedge run_rtedge_script.rt -s1024**

Creates a script named run_rtedge_script.rt that will run rtedge and create a pix file of the current screen.

====
