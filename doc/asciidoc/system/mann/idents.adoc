== NAME
idents - write a region identity summary table to a file

== SYNOPSIS

[source]
----
idents file object(s)
----


[[description]]
== DESCRIPTION

The *idents* command traverses the hierarchy of one or more `objects` and writes a summary of all regions found to `file`. The file is opened in write mode and overwritten if it already exists.

*idents* is one of four related commands that all share the same underlying logic but produce different levels of detail:

[cols="1,4"]
|===
|Command |Output detail

|`idents`
|Region-level summary only: region count, ident, air code, material code, LOS, and path. No primitives listed.

|`regions`
|Region-level summary with the Boolean formula (member names and operators) for each region.

|`solids`
|Full summary down to primitives: same as `regions`, plus the evaluated parameters of each primitive shape with accumulated transformation matrices applied.

|`tables`
|Not a valid standalone command. Invoking `tables` directly returns an input error. Use one of the three specific commands above.

|===

=== Output format

The output file is a fixed-width text table. Each region entry includes the following fields in order:

. *Region count* — sequential index of the region as encountered.
. *Region ID* (`ident`) — the region's identifier number.
. *Air code* — air region code (0 if not an air region).
. *Material code* (`GIFTmater`) — GIFT material code.
. *LOS* — line-of-sight percentage (0–100).
. *Path* — the full path from the listed object to the region.

A header is written at the top of the file containing the command name, database filename, username, database title, units, and the list of objects processed.

For `idents`, the table is written *twice*: first in the order regions are encountered during the tree walk, and then sorted by region ID number. The two listings are separated by sentinel lines (`* 9999999`).

=== Union-only traversal for idents

When building the `idents` table, non-union members of combinations (subtracted or intersected) are *skipped* during tree traversal. This avoids listing region IDs that would not appear along a shotline, preventing misleading entries in the association table.

[[examples]]
== EXAMPLES

.Write an idents summary for two groups
[example]
====
mged> **idents regions_file group1 group2**

Writes a region identity summary for all regions reachable from `group1` and `group2` to the file `regions_file`. The result string reports how many regions were processed.
====

.Write a summary for a specific region
[example]
====
mged> **idents report.txt hull.r**

Writes the idents table for the single region `hull.r` to `report.txt`.
====
