== NAME
exists - The *exists* command evaluates an expression and returns 1 if true, 0 if false. It serves roughly the same purpose for objects in a BRL-CAD database that the UNIX test command serves for files on a file system, with additional features specific to geometry.

== SYNOPSIS

[source]
----
exists {expression} [expression]...
----


[[description]]
== DESCRIPTION

*exists* evaluates boolean expressions about objects in the currently open database, returning 1 (true) or 0 (false). It is modelled on Unix-style *test(1)* commands (internally based on NetBSD's test code), but tests _objects_ rather than files.

Expressions are built from <<primaries,PRIMARIES>> and <<operators,OPERATORS>>. A bare object name with no primary or operator returns true if the object exists in the database and is not a phony placeholder (same as *-N*).

On a parse error (e.g. missing operand, unmatched parenthesis) the command returns an error status and writes a diagnostic message rather than 0.


[[primaries]]
== PRIMARIES:


*-c* _object_::
Returns TRUE if _object_ exists and is a combination.


*-e* _object_::
Returns TRUE if _object_ exists. This reports only the result of `db_lookup` without further checking, and is the most basic existence test.


*-n* _object_::
Returns TRUE if _object_ exists in the directory but is a phony placeholder (its `d_addr` is `RT_DIR_PHONY_ADDR`, meaning it has been added to the directory but not yet written to disk). Ordinary objects are never null.


*-p* _object_::
Returns TRUE if _object_ exists and is a geometric primitive (i.e. it has the solid flag set and is not a non-geometry object). Empty geometric primitives and 2D primitives like *sketch* pass this test.


*-v* _object_::
Returns TRUE if _object_ exists and its axis-aligned bounding box has non-zero volume. A sketch primitive (2D) does not pass this test; neither does a missing object.


_object1_ *=* _object2_::
Returns TRUE if both objects exist and the serialised geometry bodies of both objects are identical (same length and same bytes). The object name is excluded from the comparison, so a direct copy of an object using *cp* will compare as equal.


_object1_ *!=* _object2_::
Returns TRUE if both objects exist and their serialised geometry bodies differ (different length or at least one differing byte).


_object1_ *<* _object2_::
Returns TRUE if both objects exist and the size (in bytes) of the serialised geometry body of _object1_ is less than that of _object2_.


_object1_ *>* _object2_::
Returns TRUE if both objects exist and the size (in bytes) of the serialised geometry body of _object1_ is greater than that of _object2_.


_object1_ *-beq* _object2_::
Returns TRUE if both objects exist and the volumes of their bounding boxes are equal (within floating-point tolerance).


_object1_ *-bne* _object2_::
Returns TRUE if both objects exist and the volumes of their bounding boxes are not equal.


_object1_ *-bgt* _object2_::
Returns TRUE if both objects exist and the bounding-box volume of _object1_ is strictly greater than that of _object2_.


_object1_ *-bge* _object2_::
Returns TRUE if both objects exist and the bounding-box volume of _object1_ is greater than or equal to that of _object2_.


_object1_ *-blt* _object2_::
Returns TRUE if both objects exist and the bounding-box volume of _object1_ is strictly less than that of _object2_.


_object1_ *-ble* _object2_::
Returns TRUE if both objects exist and the bounding-box volume of _object1_ is less than or equal to that of _object2_.




[[operators]]
== OPERATORS:

primaries can be combined with the following operators:


*!* _expression_::
True if _expression_ is false.


_expression_ *-a* _expression_::
The and operator - TRUE only if both expressions are true.


_expression_ *-o* _expression_::
The or operator - true if either expression is true.


*(* _expression_ *)*::
Evaluates to true if the expression inside the parentheses evaluates to true. Used to establish order of operations.



The *-a* operator has higher precedence than the *-o* operator.


[[a_note_on_expressions]]
== A Note on Expressions

All primaries and operands must be expressed as separate arguments to exists. In practice, this means that there must be a space between each element in an exists expression. E.g. !(-c object1.s -a (object1.s -beq object2.s)) must be written as:

! ( -c object1.s -a ( object1.s -beq object2.s ) )

While this may seem like a rather verbose way of writing the expression, it greatly simplifies the parsing of the expression and is standard for virtually all *test* type commands.


[[examples]]
== EXAMPLES

The following are run from the MGED command prompt:

.Check whether an object exists
[example]
====
....
mged> exists object1.s
1
....
====

.Check if an object is a combination
[example]
====
....
mged> exists -c object1.r
1
....
====

.Use logical operators
[example]
====
....
mged> exists -c object1.r -a -e object1.s
1
....
====

.Compare bounding-box volumes
[example]
====
....
mged> exists big_sphere.s -bgt small_sphere.s
1
....
====


[[diagnostics]]
== DIAGNOSTICS

Returns 1 (true) or 0 (false) on success, and writes a diagnostic message to the result string while returning an error status on parse failures (missing operands, unmatched parentheses, or unexpected tokens).
