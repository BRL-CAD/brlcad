NAME

Command - create and manipulate a Command widget.

SYNOPSIS

Command pathName ?options?

INHERITANCE

itk::Archetype <- itk::Widget <- iwidgets::Labeledwidget <- iwidgets::Scrolledwidget <- iwidgets::Scrolledtext <- Command

Command-Line Switch:

Class:
Cmd_prefix

Command-Line Switch:
-cmd_prefix

Specifies the command prefix. If non-empty and the object specified by the cmd_prefix has a command that matches the first word of the command string, the cmd_prefix is prepended to the command and executed. Otherwise the command is executed as is.

DESCRIPTION

The Command command creates a new window (given by the pathName argument) and makes it into a command window. Additional options, described above, may be specified on the command line or in the option database to configure aspects of the command window widget. The Command command returns its pathName argument. At the time this command is invoked, there must not exist a window named pathName, but pathName's parent must exist.

METHODS

The Comand command creates a new Tcl command whose name is pathName. This command may be used to invoke various operations on the Command widget. It has the following general form:

pathName option ?arg arg ...?

Option and the args determine the exact behavior of the command. The following commands are possible for Command widgets:

BUILT-IN METHODS

isa info

ASSOCIATED METHODS

Methods associated with LIBBU's command history object:
add curr history next prev

Methods associated with the Tk text widget:

bbox compare debug delete dlineinfo get index insert mark scan search see tag window xview yview

INHERITED METHODS

Methods inherited from Archetype:

Methods inherited from iwidgets::Scrolledtext:

childsite clear export import

WIDGET-SPECIFIC METHODS

pathName edit_style [emacs | vi]
Set/get the edit style.

pathName history
Return the history list.
