Sample XMGED Displays

click on each image for a larger display

This picture shows the main display. Buttons run down the left. A command input window with scrolling, history, command line editing, and aliases is at the bottom. The drawing area in this example is embedded into the interface above the command window. Menus appear along the top as appropriate for the current state (e.g. solid and object edit menus).

This picture shows the multiview capability of XMGED. By default three ortho views and a 45,45 are shown. The lower right window is the main display (the one that will remain when multiviews is exited). Views can be swapped between windows. During editing, translations, rotations, and scales are active in all windows simultaneously.

This picture shows an entire workstation screen. Here the the drawing area is a separate window from the command window. Any number of such windows can be opened. This mode is normally used when two workstations are available side by side. For example, it is common at ARL to dedicate an SGI display for drawing and use a Sun or X Terminal for the user interface portion. Sliders, input and help windows are also shown.

This picture is of a drawing area. Multiple drawing areas can be opened on different workstations using the openw command or closed using the closew command. Also, in the upper right corner the view axis is being displayed.

This picture shows xmged while editing a tgc. The appropriate edit menus are available from the menu bar. The view axis can be seen in the lower right corner and the edit axis in the relative center. Also shown is a window containing the edit information.

This picture shows the window pane partially raised, increasing the size of the text area.
