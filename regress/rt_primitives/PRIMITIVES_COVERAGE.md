# BRL-CAD Primitive Coverage

Cross-reference of every entry in the `rt_functab OBJ[]` table
(`src/librt/primitives/table.cpp`) against what the `rt_primitives`
regression tool requires.

## What the tool gates on

1. **Candidate** — a functab slot with both `ft_prep` and `ft_shot`.  These
   are the raytraceable primitives the tool cares about.
2. **Coverage** — each candidate must have at least one object of its own
   minor type in `share/db/primitives.g` (built from `db/primitives.asc`).
   A candidate with no object is `MISSING` and **fails** the test.
3. **Validity** — each present solid must `gettree` + `prep` + shoot without
   error (`GETTREE_FAIL` / `PREP_FAIL` / `SAMPLE_FAIL` all **fail**).
4. **Skip-list** — candidates that cannot be represented or raytraced in a
   portable regression database are `EXCLUDED` (reported, never fail).  Each
   carries a one-line reason in the tool (`skip_list[]` in
   `rt_primitives.c`).
5. **Stub-shot** — pseudo-solids whose shot never makes a volumetric hit are
   `EXPECTED_ZERO`; if one ever hits, it is flagged `UNEXPECTED_HITS`.

Rays are generated by the tool itself: for each primitive, two Sobol points
on its bounding sphere define a chord ray aimed *through* the solid
(`bn_sobol_sph_sample`, per-primitive seed → deterministic and independent
of database authoring order).  Rays/sec is informational only.

## Summary

- **39** raytraceable candidates (prep+shot present).
- **30** required + present (`OK` / `ZERO_HITS` / `EXPECTED_ZERO`).
- **9** `EXCLUDED` (skip-list): `rec, poly, bspline, sph, nmg, hf, submodel,
  annot, script`.
- Expected non-`OK` informational rows: `vol` → `ZERO_HITS`; `grip`,
  `sketch`, `joint`, `datum` → `EXPECTED_ZERO`.

## Samples added to db/primitives.asc

A real NURBS `brep` (converted from a sphere; reuses the name `brep`,
replacing the deprecated bspline that used to carry it), plus `superell.s`,
`hyp.s`, `hrt.s`, `cline.s`, `joint.s`, `datum.s`.  All were produced with
`mged` (`make` / `in` / `brep`) and serialized with `g2asc`, so the `put`
syntax is canonical.

## Issues surfaced (worth filing separately)

Pre-existing librt/converter problems the tool exposed; the skip-list keeps
the regression green until they are addressed:

- **NMG raytrace bomb:** `nirt`/`rt` on any tessellated NMG asserts in
  `libnmg/isect.c` (`NMG_GET_HITMISS` returns NULL).  Independent of this
  tool.  When fixed, drop `ID_NMG` from the skip-list and add a facetized
  NMG sample.
- **asc2g `annot` crash:** a `g2asc`-serialized annotation crashes `asc2g`
  on import (access violation).
- **asc2g dangling-leaf crash:** a comb referencing a deleted solid makes
  `asc2g` access-violate rather than warning.
- **`bspline` removed from asc2g:** importing the legacy bspline is now a
  hard error.
- **`sph`/`rec` aliasing:** both share rt_ell/rt_tgc import/export, so their
  dedicated minor types never appear on disk; their specialized shots are
  unreachable from a stored db.
- **superell debug print:** `rt_superell_shot()` has a leftover
  `bu_log("realroot ...")` (throttled to 10 lines via a `static` counter).
- **`vol` zero hits:** the bundled vol sample reports 0 hits (sparse data /
  oversized bbox); pre-existing, non-gating.

## Build-config note

`primitives.g` is built by `asc2g` (requires Tcl); when Tcl is off the `gcv`
fallback is used and may not import the serialized brep, so the CMake driver
passes `--optional brep` in that configuration to relax that one
requirement.
